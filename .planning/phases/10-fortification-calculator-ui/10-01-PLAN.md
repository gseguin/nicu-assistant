    ### String-bridge for SelectPicker binding (LOCKED)

    `state.svelte.ts` stores `targetKcalOz: TargetKcalOz` (numeric literal union), `base: BaseType` (string union), `formulaId: string`, `unit: UnitType` (string union). SelectPicker only accepts `bind:value` to a `string` local. The component declares four `$state` string mirrors and two `$effect` blocks to keep them synchronized with the state singleton:

    ```typescript
    let kcalStr = $state(String(fortificationState.current.targetKcalOz));
    let baseStr = $state<string>(fortificationState.current.base);
    let formulaStr = $state<string>(fortificationState.current.formulaId);
    let unitStr = $state<string>(fortificationState.current.unit);

    // Mirror → state (user input flows down to state)
    $effect(() => {
      fortificationState.current.targetKcalOz = parseInt(kcalStr, 10) as TargetKcalOz;
      fortificationState.current.base = baseStr as BaseType;
      fortificationState.current.formulaId = formulaStr;
      fortificationState.current.unit = unitStr as UnitType;
    });

    // State → mirrors (external state changes — e.g. init from sessionStorage, or auto-reset — flow up to mirrors)
    $effect(() => {
      const s = fortificationState.current;
      if (kcalStr !== String(s.targetKcalOz)) kcalStr = String(s.targetKcalOz);
      if (baseStr !== s.base) baseStr = s.base;
      if (formulaStr !== s.formulaId) formulaStr = s.formulaId;
      if (unitStr !== s.unit) unitStr = s.unit;
    });
    ```

    The if-equality guards prevent the two effects from ping-ponging. Each SelectPicker uses `bind:value={kcalStr}` / `bind:value={baseStr}` / `bind:value={formulaStr}` / `bind:value={unitStr}` — no `as string` casts inside the template (Svelte 5 does not support cast expressions in `bind:`).

    ### 5 inputs (rendered in this order inside a single `<section class="card flex flex-col gap-4">`)

    ### Auto-reset effect (LOCKED — the ONLY place that mutates unit)

    ```typescript
    let prevFormulaId = fortificationState.current.formulaId;
    $effect(() => {
      const currFormulaId = fortificationState.current.formulaId;
      if (
        prevFormulaId === 'similac-hmf' &&
        currFormulaId !== 'similac-hmf' &&
        fortificationState.current.unit === 'packets'
      ) {
        fortificationState.current.unit = 'teaspoons';
      }
      prevFormulaId = currFormulaId;
    });
    ```

    Properties:
    - Does NOT fire on initial mount (the condition `prev === 'similac-hmf' && curr !== 'similac-hmf'` is false when `prev === curr`).
    - Fires ONLY on the similac-hmf → non-similac-hmf transition, and only if the user had packets selected.
    - Does NOT fire on the reverse (non-HMF → HMF) transition.
    - Does NOT fire when the user changes unit directly.
    - After mutating `unit`, the state→mirrors effect re-syncs `unitStr = 'teaspoons'`, and on the next render `isBlocked` evaluates false — the inline message does NOT show for this transition.
