    // Simulate reload directly instead: write to sessionStorage, then call init().
    sessionStorage.setItem(

      JSON.stringify({ weightKg: 7.5 })

    uacUvcState.init();

    expect(uacUvcState.current.weightKg).toBe(7.5);
  });

**Why scenario 5 uses direct `sessionStorage.setItem` instead of the naive `persist → reset → init` chain:** `uacUvcState.reset()` clears sessionStorage as part of its contract (state.svelte.ts reset() lines). So the intuitive `persist → reset → init` round-trip loses the persisted value at the `reset()` step. The correct test shape is `persist → mutate in-memory → init from storage` OR `write storage directly → init`. We use the latter here to keep the assertion focused on `init()`'s behavior. This is a direct translation of GIR's own persistence test pattern (inspect `GirCalculator.test.ts` if uncertain) — it is the idiomatic shape.
