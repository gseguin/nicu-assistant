|------|---------|------|--------|-------------|-------------------|

| 2xs (eyebrow) | `text-2xs` `font-semibold tracking-wide uppercase` | 11px (0.6875rem) | 600 | 1.3 | Card eyebrows `"UAC DEPTH — ARTERIAL"` / `"UVC DEPTH — VENOUS"`, NumericInput range hint |

**Font weights used (exactly 3):** 400 (regular body/empty), 600 (semibold — eyebrows, labels, subtitle emphasis), 900 (black — hero display numerals only). Project-wide convention inherited from v1.0; not phase-specific.

**Tabular numerics:** hero values require `.num` utility (from `app.css:160-163` — `font-variant-numeric: tabular-nums; font-feature-settings: 'tnum'`). **Mandatory** for all numeric clinical output per PROJECT.md §Design Context: "tabular numerics for clinical outputs".

**Hero value decimal precision:** `toFixed(1)` — 1 decimal. **Locked** (resolves RESEARCH.md Open Question — flagged as Claude's discretion D-06 override). Rationale:
1. Matches xlsx native precision (cells B3/B7 compute to 2 decimals; clinicians dictate to 1).
2. Matches bedside catheter markings (0.5 cm / 1.0 cm intervals) — 2 decimals reads as false precision.
3. Consistent with `GirCalculator.svelte:119` (`currentGirMgKgMin.toFixed(1)`).

**Eyebrow text final form (resolves RESEARCH.md Open Question #1 / A1):**
- Primary: `"UAC DEPTH — ARTERIAL"` (UAC card) / `"UVC DEPTH — VENOUS"` (UVC card).
- Measured width at `text-2xs font-semibold tracking-wide uppercase` ≈ 115px against ≥ 300px inner card width at 375px mobile (1-col stacked layout) — **fits one line with comfortable margin**.
- **Fallback (NOT required, captured for auditor):** if a future render regression at 375px causes wrap, shorten to `"UAC — ARTERIAL"` / `"UVC — VENOUS"`. Executor MUST NOT pre-shorten without evidence of wrap at mobile in both themes.

**Em dash character:** `"UAC DEPTH — ARTERIAL"` uses the **em dash (U+2014)** with a single space on either side (matches typographic convention already shipped in GIR bucket labels per ROADMAP v1.9 POLISH-04 "bucket labels normalized to en-dash typography" — Phase 42 uses em-dash for compound-description eyebrow). Executor MUST paste the literal Unicode character, not `--` or `&mdash;`.

## Interaction Contract

### Primary interaction: weight entry

1. **Textbox path (UAC-01):** user taps textbox → `inputmode="decimal"` triggers iOS decimal keyboard → user types `2.5` → `oninput` parses and writes `uacUvcState.current.weightKg = 2.5` → `$derived(calculateUacUvc(...))` produces `{uacCm: 16.5, uvcCm: 8.25}` → both hero cards re-render → `animate-result-pulse` fires (keyed off `pulseKey = weight.toFixed(2)`) → `aria-live="polite"` regions announce the new values.
2. **Slider path (UAC-02):** user drags range thumb → `oninput` fires on every step (~97 positions across 0.3–10 kg at step 0.1) → each event writes `uacUvcState.current.weightKg = parsed` → same reactive chain as textbox path → textbox value display updates in lockstep.
3. **Bidirectional sync (UAC-02):** both surfaces read the same `$state.weightKg`. When textbox writes, slider thumb re-reads the value and visually snaps to the nearest step position. When slider writes, textbox displays the exact stepped value. **Pattern source:** RESEARCH.md §"Bidirectional Sync Pattern" — `bind:value` on NumericInput + manual `oninput` + `value={state ?? min}` on range.

### Pulse animation key

**Locked (resolves Claude's Discretion):** `pulseKey = weight.toFixed(2)`.

Rationale: `weight` is the only independent variable in Phase 42; both UAC and UVC derive from it. Keying off `weight.toFixed(2)` (not the result values) avoids redundant recomputation, is simpler to reason about, and 2-decimal precision ensures a pulse on any clinically-meaningful change (smallest step is 0.1 kg = 0.10 at 2-decimal precision). No dual-result key needed (as GIR uses for `${gir}-${rate}` because GIR has two independent metrics).

### Out-of-range advisory (UAC-07)

- **Textbox path:** user types `15` → blurs field → NumericInput's `hasBlurred = true` + `aboveMax = (15 > 10)` → `rangeError = "Outside expected range — verify"` → `<p class="... text-error">` appears below textbox with `transition:slide` (reduced-motion gated).
- **Slider path:** **cannot produce out-of-range** (HTML range input hard-clamps to `min`/`max` attributes). No advisory possible from this path.
- **During drag:** no advisory shown until blur — matches the v1.6 NumericInput no-auto-clamp contract. Slider keystrokes never focus the textbox, so slider changes never fire a textbox blur.

### Keyboard controls

- **Textbox:** standard decimal input. `Enter` blurs to confirm (NumericInput.svelte:84-86). `Tab` / `Shift+Tab` move focus to slider / previous control.
- **Slider (native `<input type="range">`):** `ArrowLeft`/`ArrowRight` step by `±step` (±0.1 kg). `Home`/`End` jump to `min`/`max`. `PageUp`/`PageDown` step by ±10×`step` (±1.0 kg). All behaviors inherited from browser native; no manual keydown handlers needed.
- **Focus order:** `<body>` → nav → header → NumericInput textbox → range slider → (if visible) AboutSheet trigger (lives outside the calculator component in the shell header). Reading order matches visual order.

### Persistence

- sessionStorage key: `nicu_uac_uvc_state` (CONTEXT.md D-09). Shape: `{ weightKg: number | null }`.
- Writes on every `uacUvcState.current` mutation via `$effect(() => { JSON.stringify(uacUvcState.current); uacUvcState.persist(); })` inside `UacUvcCalculator.svelte` (mirrors `GirCalculator.svelte:36-39`).
- Init on route `onMount` via `uacUvcState.init()` in `+page.svelte` (mirrors `/gir`).
- Private-browsing / quota-exceeded failures are silent (RESEARCH.md Pitfall #6).

### `aria-live` regions

- Each hero card's outer `<section>` carries `aria-live="polite"` and `aria-atomic="true"` (UAC-06 + CONTEXT.md D-06).
- Atomic regions mean screen readers announce the entire card content on change, not a diff — this is correct because the eyebrow + value + unit should be read together as one statement (`"UAC DEPTH — ARTERIAL, 16.5, cm"`).
- **No duplicate aria-live** on the pulse wrapper; the `{#key pulseKey}` is purely a Svelte re-mount trigger for the animation and carries no ARIA semantics.

### Non-favorited active-route indication (Phase 41 D-03 obligation)

UAC/UVC is non-favorited by default → NavShell's bottom bar / top nav will NOT highlight any tab when the user is on `/uac-uvc`. The route's own `<h1>` + Ruler icon in the header IS the active-route indicator. **This UI-SPEC locks:**
- h1 text: `UAC/UVC Catheter Depth` (not just `UAC/UVC` — the fuller phrase makes standalone identification unambiguous).
- Icon color: `text-[var(--color-identity)]` (rose) — visually asserts "this is the UAC/UVC route" independently of nav state.
- Subtitle contextualizes the unit and formula kind.

If a clinician favorites UAC/UVC, NavShell lights up the tab underline via the existing identity machinery — zero additional UI work. Header h1 + icon redundantly reinforce the route identity in both favorited and non-favorited states.
