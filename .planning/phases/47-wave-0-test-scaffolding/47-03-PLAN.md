// Phase 47 / TEST-03: smoke check that the webkit-iphone Playwright project
// is wired correctly. Behavioral assertions belong to Phase 49 (DRAWER-TEST-03).
//
// Goal: prove (a) the project loads, (b) the page navigates, (c) the running
// browser reports a non-null window.visualViewport (real WebKit + real chromium,
// independent of the jsdom polyfill from Plan 47-01). This single test runs in

import { expect, test } from '@playwright/test';

test('visualViewport is defined', async ({ page }) => {
  await page.goto('/');
  const hasVV = await page.evaluate(() => {

});

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI workflow → external Playwright CDN | The `playwright install` step downloads ~80 MB of webkit binary from Playwright's CDN. This is the same trust boundary as the existing chromium install — same package, same provenance, same `--with-deps` system-library install. No new boundary introduced. |
| Smoke spec → running browser engine | `e2e/webkit-smoke.spec.ts` evaluates JS in the Playwright-controlled browser context. The browser is sandboxed; `page.evaluate(...)` cannot escape to the Playwright host. No new boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-47-09 | Tampering | `playwright.config.ts` `chromium` entry accidentally modified | mitigate | Task 1's done criterion explicitly requires the existing `chromium` entry to remain byte-for-byte identical. The chromium-suite-green gate (`pnpm exec playwright test --project chromium` reports 99/99) at the verify step is the regression sentinel — if the chromium entry is mutated (e.g. wrong `devices['Desktop Chrome']` spread, accidental `headless: false`), at least one chromium spec will surface a delta. |
| T-47-10 | Tampering | `.github/workflows/ci.yml` other lines accidentally modified | mitigate | Task 2's done criterion requires the `git diff` to show ONLY the one-line browser-list change. The verify step grep-filters the diff for the playwright install line and asserts the change is minimal. |
| T-47-11 | Information disclosure | Smoke spec leaks user data | n/a | The spec navigates to `/` (no auth, no PII), evaluates a typeof check, and asserts `window.visualViewport` is defined. No sensitive data crosses the test boundary. |
| T-47-12 | Denial of service | webkit binary download fails on CI | accept | Playwright's CDN has high uptime; if it's down, the `Install Playwright browsers` step fails and CI is red. This is a transient infrastructure failure, not a Phase 47 design flaw. The fix (re-run the job) is built into GitHub Actions. The `--with-deps` flag has a slow but reliable retry semantics. |
| T-47-13 | Elevation of privilege | New webkit binary on CI runner has elevated capabilities | accept | The CI runner is `ubuntu-latest`, ephemeral, with no persistent state. Adding a second browser binary doesn't elevate privileges any more than adding chromium did. Same threat model as the existing chromium install. |
| T-47-14 | Repudiation | Existing chromium 99-passing suite regresses unnoticed | mitigate | D-18 + Task 1's verify step `pnpm exec playwright test --project chromium` confirms 99/99 chromium pass. The CI log records the per-project pass count. Any chromium regression surfaces immediately (CI red). |
| T-47-15 | Information disclosure | webkit-iphone project runs ALL specs (D-15) and a spec leaks calculator state | accept | Existing specs were designed for chromium; running them under webkit-iphone may surface different timing or DOM behavior, but they don't expose new attack surface. The deferred follow-up to migrate or skip incompatible specs is informational, not a security gate. |
