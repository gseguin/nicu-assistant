bits-ui Select API (verified 2.16.5):
- Select.Root type="single" bind:value items={options}
- Select.Trigger — button that opens the dropdown
- Select.Portal — renders content outside component tree (fixes z-index stacking)
- Select.Content preventScroll={true} — locks body scroll when open
- Select.Viewport class="overscroll-contain" — prevents iOS overscroll propagation
- Select.Group — wraps grouped options
- Select.GroupHeading — group label
- Select.Item value label — individual option
- Arrow Up/Down, Home/End, Enter, Escape — all handled automatically by bits-ui

  <action>
    Create src/lib/shared/components/SelectPicker.svelte (create the components/ subdirectory if it does not exist).

    Component props interface:
    ```typescript
    let {
      label,
      value = $bindable(),
      options,
      placeholder = 'Select...',
      class: className = '',
    }: {
      label: string;
      value: string;
      options: SelectOption[];
      placeholder?: string;
      class?: string;
    } = $props();
    ```

    Derived state:
    - groups: unique group names from options where group is defined
    - hasGroups: groups.length > 0
    - selectedLabel: label of the option matching value, or placeholder if none

    Context:
    - Call getCalculatorContext() to get accentColor for the group heading color
    - Do NOT pass accentColor as a prop (per D-06, context is the pattern)

    Structure:
    ```
    Select.Root type="single" bind:value items={options}
      Select.Trigger — displays selectedLabel, aria-label="{label}: {selectedLabel}"
      Select.Portal
        Select.Content preventScroll={true} — styled as bottom sheet on mobile, floating on desktop
          Select.Viewport class="overscroll-contain max-h-[70svh] overflow-y-auto"
            {#if hasGroups}
              {#each groups as group}
                Select.Group
                  Select.GroupHeading — sticky, uses accentColor from context
                  {#each options filtered to this group}
                    Select.Item value label
                {/each}
            {:else}
              {#each options}
                Select.Item value label
    ```

    Styling requirements (all colors via CSS custom properties — NEVER hardcoded oklch()):
    - Trigger: min-h-[2.75rem] (44px minimum touch target, WCAG), w-full, rounded-lg
    - Trigger background: bg-[var(--color-surface-card)]
    - Trigger border: border border-[var(--color-border)]
    - Trigger text: text-[var(--color-text-primary)]
    - Content panel: rounded-2xl bg-[var(--color-surface-card)] shadow-2xl
    - Group heading: text-2xs font-bold uppercase tracking-[0.18em], color = accentColor from context
    - Group heading: sticky top-0 bg-[var(--color-surface-card)] (prevents heading scrolling under items)
    - Items: rounded-xl px-4 py-3 min-h-[44px] text-[var(--color-text-primary)]
    - Selected item indicator: checkmark icon from @lucide/svelte using accentColor
    - Dark mode: all colors are automatic via CSS custom properties (no dark: classes needed on individual values)

    Critical pitfalls to avoid (from RESEARCH.md):
    - Do NOT manually focus() items inside Select.Viewport (Pitfall 1 — bits-ui uses combobox pattern, trigger keeps focus)
    - Do NOT use lucide-svelte — use @lucide/svelte (already installed)
    - Do NOT use 'id' as option key — use 'value' (per D-02 and anti-patterns in RESEARCH.md)
    - MUST add overscroll-contain on Select.Viewport (Pitfall 7 — iOS scroll propagation)
    - Do NOT add document.body.style.overflow manipulation — preventScroll={true} handles it
    - MUST use var(--color-*) for every color — no hardcoded oklch() anywhere
