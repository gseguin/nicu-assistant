# Phase 41 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** Phase 41 introduces **zero new visual primitives**. It flips the render source of the existing mobile bottom bar and desktop top nav from `CALCULATOR_REGISTRY` → `favorites.current`-filtered list. The design system, spacing, typography, color, and copywriting contracts are all **preserved unchanged** from the v1.4 shell + v1.5 identity system + v1.12 shipped state. This UI-SPEC locks those contracts explicitly so no drift is introduced during the render-source flip.

## Color

All values are existing OKLCH tokens in `app.css`. No new palette decisions in this phase.

| Role | Value (light) | Value (dark) | Usage in this phase |
|------|---------------|--------------|---------------------|
| Dominant (60%) — title bar + bottom bar surface | `var(--color-surface)` = `oklch(97.5% 0.006 225)` | `oklch(16% 0.012 240)` | `bg-[var(--color-surface)]` on `<header>` and bottom `<nav>` |
| Secondary (30%) — border under title bar / above bottom bar | `var(--color-border)` = `oklch(63% 0.01 220)` | `oklch(58% 0.018 235)` | `border-b` on header, `border-t` on bottom nav |
| Accent (10%) — **per-tab identity hue** (existing `--color-identity` set by `identityClass`) | varies per calculator (Morphine clinical 220, Formula teal ~195, GIR dextrose green ~145, Feeds terracotta ~30) | per-calculator OKLCH pair | `text-[var(--color-identity)]` on active tab label + icon; `border-[var(--color-identity)]` bottom border on desktop active; `outline-[var(--color-identity)]` on focus |
| Destructive | `var(--color-error)` | `var(--color-error)` | **Not used** in Phase 41 (no delete/confirm surfaces) |
| Inactive tab text | `var(--color-text-secondary)` | `var(--color-text-secondary)` | Unselected tab label |
| Hover tab text | `var(--color-text-primary)` | `var(--color-text-primary)` | Cursor-hover on inactive tab (desktop only; mobile has no hover state) |

**Accent reserved for:** active-tab label text + icon, active-tab indicator (bottom border desktop, font-weight + color mobile), focus-visible outlines. **Never** used for inactive tabs, hamburger button, theme button, or non-nav surfaces in Phase 41.

**Identity hue coverage by favorites:** Each favorited tab wears its own `--color-identity` derived from `identityClass` on the registry entry. The render flip from `CALCULATOR_REGISTRY` → favorites-filtered list preserves the per-tab `identityClass` verbatim — the class travels with the entry, so identity colors "just work" after the flip.

**Contrast requirement (locked):** All four shipped identity hues already pass 4.5:1 contrast vs `--color-surface` in both themes (v1.5 Morphine retune + v1.8 GIR + v1.12 Feeds axe sweeps are green). Phase 41 MUST NOT alter any identity-hue token values. If a FAV-TEST-04 axe sweep ever flags a contrast regression, the fix belongs in the identity-hue source, not in Phase 41's render code.

**Hands-off for Phase 41:** No color token additions, no color class changes in `NavShell.svelte`'s tab markup beyond what's needed to pass the favorites-filtered iterand.

## Interaction Contract (added section — Phase 41 is primarily an interaction flip)

| Interaction | Locked behavior | Source / evidence |
|-------------|----------------|-------------------|
| User taps/clicks a favorited tab | Navigate to `{calc.href}` via `<a>` — existing shell behavior | `NavShell.svelte` mobile+desktop `<a>` elements |
| User on a non-favorited route | All rendered tabs show inactive state (no `aria-selected="true"`, no identity color, no border) | NAV-FAV-03 + CONTEXT.md D-03 |
| User un-favorites current route from hamburger | Bar silently removes that tab; URL unchanged; no redirect; focus returns to hamburger button on close | CONTEXT.md D-08 + Phase 40 D-02 |
| User re-favorites a previously-un-favorited calculator | Tab slots back into registry-order position (not appended) | CONTEXT.md D-07 + Phase 40 D-07 + `favorites.toggle()` in `favorites.svelte.ts` |
| User has zero favorites | Bars render empty; hamburger remains reachable in title bar | CONTEXT.md D-02 |
| First render before `favorites.init()` completes | Default set shown synchronously (module-scope seed) — no empty→default flash | CONTEXT.md D-07 |
| Focus visible state | `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-[var(--color-identity)]` (desktop) / `-offset-[-2px]` (mobile) — existing v1.4 pattern | `NavShell.svelte` current markup |
| Reduced-motion | No new animations added — existing transitions (`transition-colors`) honor `prefers-reduced-motion` via Tailwind's globals | existing v1.6/v1.10 pattern |

## Layout Invariants (locked from v1.4 shell)

The following structural properties of `NavShell.svelte` MUST NOT change in Phase 41:

1. **Title bar** stays as the sticky top element (`sticky top-0 right-0 left-0 z-10`). Hamburger left, app name, optional desktop tabs, spacer, theme button right. The hamburger button is already in place from Phase 40.
2. **Mobile bottom bar** stays `fixed right-0 bottom-0 left-0 z-10`, hidden on `md+` (`md:hidden`). `min-h-14` per tab, `pb-[env(safe-area-inset-bottom,0px)]` on the `<nav>` wrapper.
3. **Desktop top nav** stays inside the title bar `<header>`, hidden on mobile (`hidden md:flex`), with `ml-4` from the app name and `gap-2` between tabs.
4. **Role semantics** stay as `role="tablist"` > `role="tab"` + `aria-selected={boolean}` (NOT migrated to `role="navigation"` / `aria-current="page"` — see CONTEXT.md D-04).
5. **Per-tab identity** comes from `{calc.identityClass}` applied on the `<a>` element so `--color-identity` resolves inside the scope. The render-source flip keeps this exactly as-is.

**What Phase 41 IS allowed to touch inside `NavShell.svelte`:**
- Replace the iterand in both `{#each CALCULATOR_REGISTRY as calc}` blocks with the new favorites-filtered iterand (CONTEXT.md D-01).
- Add a `$derived` computing the visible entries.
- Replace the hardcoded `activeCalculatorId` ternary with a registry-driven lookup (CONTEXT.md D-05) — this is a Wave-0 cleanup inside a file already being edited.
