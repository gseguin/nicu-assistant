## Canonical Reference

This phase's design contract is **`/mnt/data/src/nicu-assistant/DESIGN.md`** at the project root. Every visual decision in this phase honors DESIGN.md's named rules:

- **The Identity-Inside Rule** — identity hue scoped to the calculator's own surfaces; never in chrome
- **The Amber-as-Semantic Rule** — amber reserved for BMF fortifier mode in the formula calculator
- **The OKLCH-Only Rule** — all colors declared in OKLCH; no hex/RGB/HSL anywhere
- **The Red-Means-Wrong Rule** — red reserved for errors and out-of-range advisories
- **The Five-Roles-Only Rule** — Plus Jakarta Sans at exactly five sizes (display/title/body/ui/label)
- **The Tabular-Numbers Rule** — every clinical number applies the `.num` utility
- **The Eyebrow-Above-Numeral Rule** — single label-role eyebrow above the hero numeral
- **The 11 px Floor** — `text-2xs` (11 px) is for labels only; advisory copy steps up to `text-ui` (13 px)
- **The Tonal-Depth Rule** — depth via three surface tokens, not shadows
- **The Flat-Card-Default Rule** — flat 1 px-bordered card with `shadow-sm`; never nested

The `<HeroResult>` reference implementation (HTML + CSS) lives in **`/mnt/data/src/nicu-assistant/DESIGN.json`** under `components[].name = "Hero Result Card"`. Implementations match within token tolerance; if a value is needed and not in the token system, add a token in `app.css` — never inline a hex.

## New Components (must be designed)

### `<HeroResult>` — `src/lib/shared/components/HeroResult.svelte`

**Purpose.** Single shared hero pattern for every calculator's primary result. Replaces the 5 divergent hero shapes shipping today (morphine multi-cell summary chip, formula single mint chip without `.num`, GIR two-line custom hero, feeds bedside-mode no-hero, uac-uvc paired custom hero).

**Props (Svelte 5 runes):**

```typescript
{
  eyebrow: string;             // UPPERCASE label (e.g., "ARTERIAL DEPTH")
  value: string;               // already-formatted display string (e.g., "16.5")
  unit?: string;               // suffix (e.g., "cm", "mg/kg/dose")
  secondary?: string | Snippet; // optional ui-size line below (e.g., "Shukla / Dunn")
  pulseKey?: string | number;  // drives {#key} re-mount for animate-result-pulse
  icon?: ComponentType;        // optional lucide icon (D-05 third-cue support)
  ariaLabel?: string;          // override for accessible label
  children?: Snippet;          // optional override of default body (used by morphine 3-cell summary, GIR two-line)
}
```

**Visual contract** (matches DESIGN.md §5 Hero Result + DESIGN.json `components[].name = "Hero Result Card"`):

- **Card chrome:** `class="card hero-card animate-result-pulse"`, identity-tinted background `style="background: var(--color-identity-hero);"`, 1 px `--color-border`, `shadow-sm`, `rounded-2xl`/`rounded-3xl`, padding `1.5rem 1.25rem`.
- **Eyebrow:** `text-2xs font-semibold uppercase tracking-[0.02em] text-[var(--color-identity)]`, single line. Optional `icon` prop renders to the LEFT of the eyebrow at `size={24}`, `class="text-[var(--color-identity)]"`, `aria-hidden="true"`.
- **Numeral:** `class="num text-display font-extrabold text-[var(--color-text-primary)]"` — the `.num` utility (tabular-nums) is **mandatory** per The Tabular-Numbers Rule.
- **Unit:** `text-ui font-medium text-[var(--color-text-secondary)]`, baseline-aligned beside the numeral via `flex items-baseline gap-2`.
- **Secondary line:** `text-ui text-[var(--color-text-secondary)]` (13 px — never 11 px per The 11 px Floor).
- **ARIA:** outer `<section aria-live="polite" aria-atomic="true">` always.
- **Motion:** `animate-result-pulse` (200 ms entrance) re-fires on `pulseKey` change via `{#key pulseKey}`. Reduced-motion gate inherited from `app.css:180-184`.
- **Em-dash ban:** no `—` or `--` in any rendered eyebrow/secondary/aria copy. Use `·` (middle-dot, U+00B7), `:`, `;`, or parentheses (per `917ecf2`).

**Component test scenarios (≥ 5):**
1. Renders eyebrow + value + unit in correct visual hierarchy
2. `.num` class applied to value
3. `aria-live="polite"` + `aria-atomic="true"` present on outer container
4. `pulseKey` change re-mounts inner block (animation re-fires)
5. `children` Snippet override path replaces default body (used by morphine + GIR)
6. `icon` prop renders at `size={24}` with identity color, `aria-hidden="true"`
7. `prefers-reduced-motion: reduce` honored (animation disabled)

### `<DisclaimerBanner>` — `src/lib/shared/components/DisclaimerBanner.svelte`

**Purpose.** Replace the blocking disclaimer modal (`DisclaimerModal.svelte`) with a dismissable inline banner per the warm-clinical Don't-default-to-modal principle.

**Props:**

```typescript
{
  onShowFull?: () => void;  // opens AboutSheet at the Disclaimer row
}
```

State is read internally from `src/lib/shared/disclaimer.svelte.ts` (singleton). Acknowledgment persists via `localStorage['nicu_assistant_disclaimer_v2']`.

**Visual contract:**

- **Position:** 1-line banner pinned below the title bar, above the page content slot in `+layout.svelte`. Mounted once at the layout level (not per-route).
- **Surface:** `bg-[var(--color-surface-alt)]` with 1 px bottom `--color-border`. Dark mode inherits via the same token.
- **Body copy:** `text-ui` (13 px) weight 500, `text-[var(--color-text-primary)]`. Single sentence, period-terminated, no em-dash. Provisional copy: `"Clinical decision support, not medical advice. Verify all values before clinical use."` (legal review checkpoint per RESEARCH risk #2).
- **"More" link:** inline `<button type="button" class="underline">More</button>` opens AboutSheet at the new Disclaimer row.
- **Dismiss button:** right-aligned `.icon-btn` (48 × 48 hit area) carrying `<X size={18} aria-hidden="true" />` with `aria-label="Dismiss disclaimer"`.
- **ARIA:** outer `<div role="region" aria-label="Clinical use disclaimer">`.
- **Motion:** **no animation on first paint** (instant render — the banner must not feel like a delayed nag). Slide-out 200 ms ease-out on dismiss, gated on `prefers-reduced-motion`.

**Persistence + migration (per RESEARCH finding 1, correcting CONTEXT D-14):**

- The codebase already ships key `nicu_assistant_disclaimer_v1` (`src/lib/shared/disclaimer.svelte.ts:3`), NOT bare `nicu_assistant_disclaimer` as CONTEXT.md states.
- `init()` reads BOTH `_v2` and `_v1` keys: if either is `'true'`, treat as acknowledged. If only `_v1` is set, write `_v2` for forward compatibility.
- `acknowledge()` writes `_v2` only.
- localStorage write failures (private browsing) swallow silently; in-memory acknowledgment still works.

**Component test scenarios (≥ 5):**
1. Fresh install: banner visible, not acknowledged
2. Migration: `_v1='true'` only → acknowledged on init, `_v2` written, banner hidden
3. `_v2='true'` already: acknowledged, banner hidden, no `_v1` read
4. Dismiss button (click + keyboard activation) acknowledges, persists `_v2`
5. "More" link calls `onShowFull` → opens AboutSheet at Disclaimer row
6. localStorage write throws (private mode) → in-memory acknowledge still works, no crash

## Acceptance Criteria — Visual

For each calculator route, post-42.1:

1. **Hero is the largest object on the screen.** The display numeral renders at `text-display` (44 px / weight 800 / `.num`) inside an identity-tinted card with the eyebrow (UPPERCASE, identity color, label role) above and the unit suffix (body weight, `--color-text-secondary`) baseline-aligned beside the numeral. Optional secondary line at `text-ui` (13 px) below.
2. **(D-08, deferred to its own plan + localStorage flag per RESEARCH risk #1)** On mount, hero card occupies above-the-fold viewport with pre-computed defaults; inputs collapse to a sticky drawer above the bottom nav. Gated behind `localStorage['nicu_assistant_drawer_ux'] = 'on'` for internal QA before flipping default — the planner can ship Plan 5 with the flag OFF.
3. **Mobile bottom nav does not occlude page content** at 375 × 667 (iPhone SE / 14 mini) or 414 × 896 (iPhone 14 Pro Max). Verified by Playwright `e2e/mobile-nav-clearance.spec.ts` — last content element's `getBoundingClientRect().bottom` < bottom-nav `top` after scroll-to-bottom.
4. **Active calculator state in chrome is Clinical Blue, NOT the identity hue.** Top-tab underline uses `--color-accent`; bottom-tab text and inset stripe use `--color-accent`. Identity hue appears only inside calculator surfaces (eyebrow, hero tint, slider track, calculator-input focus ring).
5. **Hamburger drawer star icons are neutral.** Filled (favorited) = `--color-text-primary`; outlined (not favorited) = `--color-text-tertiary`. No identity color in the drawer.
6. **No em-dashes** (`—`, `--`, or `&mdash;`) appear in any rendered string, `aria-label`, or screen-reader copy across the app.
7. **No side-stripe borders** (`border-l-N`, `border-r-N`, `border-t-N`, `border-b-N` with N ≥ 2 as a colored accent) on any card, list item, or alert.
8. **No nested cards.** Period.
9. **No glassmorphism, ambient glows, or decorative shadows** anywhere except the single bottom-nav `bg-[var(--color-surface)]/95 backdrop-blur` license.
10. **No scroll-driven animation** anywhere in the app — dock magnification removed from both Morphine and GIR.
11. **axe sweep returns 0 violations** in light AND dark mode for every calculator route, the disclaimer banner (visible state), and the AboutSheet at the new Disclaimer row.
12. **Screen reader announces the hero result** when the value changes via `aria-live="polite"` + `aria-atomic="true"` on the outer `<section>`.
