## Visual description of the morphine prototype (for user review)

When you open `/morphine-wean`, you see:

**Mobile (<768 px) — vertical stack:**
1. Top title bar (`NICU Assist` + hamburger + theme toggle).
2. Disclaimer banner (if not yet acknowledged).
3. Page header `[Syringe icon] Morphine Wean` (`text-title font-bold`).
4. **Hero card** — `WEANING SUMMARY` eyebrow (identity-blue, uppercase) above a 3-cell strip showing `Start 0.124 mg → Step 10 0.012 mg | Total reduction 90.00%`. Identity-tinted background, animate-result-pulse on recalculation. Shows real numbers immediately on first paint because morphine state seeds from `morphine-config.json` defaults.
5. 10-row weaning schedule below the hero (each row tappable to set `aria-current="step"` + identity inset ring).
6. Sticky **inputs drawer handle** pinned above the bottom nav. Reads:
   ```
   INPUTS  Weight 3.1 kg · 10% step       [ChevronUp]
   ```
   Background: surface-card 95% with `backdrop-blur`. `min-h-14`, focus-ring `--color-accent`. Sits at `bottom: calc(env(safe-area-inset-bottom, 0px) + 4rem)` so it clears the bottom nav.
7. Mobile bottom nav (Plan 1 unchanged).

**Tap the handle** → bottom-sheet `<dialog>` slides up to ~80dvh, slide animation gated on `prefers-reduced-motion`. Sheet header: `Inputs` title + chevron-down close button. Body: the 3 NumericInput fields + Clear inputs button. Tap-outside or Esc → drawer collapses, hero re-fills the viewport.

**Desktop (≥768 px) — two-column grid:**
- Left column (`minmax(0, 1fr)`): hero card + 10-row schedule.
- Right column (`22rem`): inputs card sticks to `top-20` as user scrolls. The mobile drawer handle is `md:hidden`, the bottom-sheet `<dialog>` never opens because there's no trigger.

Defaults give you the hero numerals (start 0.124 mg, end 0.012 mg, 90.00% reduction) on first paint, satisfying D-08's "result is the largest object on screen at 0 ms after hydration" criterion.
