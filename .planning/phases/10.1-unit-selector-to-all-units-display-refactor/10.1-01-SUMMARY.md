# Phase 10.1 Plan 01: Unit Selector to All-Units Display Refactor Summary

Removed the Unit SelectPicker from the Fortification calculator and now display all applicable unit amounts (grams, teaspoons, tablespoons, scoops, packets-when-HMF) simultaneously, with verification computed from the grams branch.

## Tasks Completed

1. **Task 1 — State cleanup:** Dropped `unit` field from `FortificationStateData`, `MockFortificationState`, and the corresponding state tests. `UnitType` import removed from state files (still imported by component for `ALL_UNITS`).
2. **Task 2 — Locked reference values:** Ran `calculateFortification` against the documented Neocate parity case for all 5 units at volume=180 and volume=360. Recorded verbatim in a comment block at the top of `FortificationCalculator.test.ts`.
3. **Task 3 — Component refactor:** Removed Unit SelectPicker, `unitStr` mirror + its 2 sync effects, `isBlocked` derived, `prevFormulaId` tracker, auto-reset effect, the inline "Packets unavailable" message, and the Suggested Starting Volume display. Added `ALL_UNITS` const, `unitResults` derived (5 calls per recalc), `gramsResult` derived, `shouldShowRow` helper, and an all-units `<dl>` template with `data-unit-row={unit}` attributes. Verification card now reads exclusively from `gramsResult`.
4. **Task 4 — Test refactor:** Deleted Tests 4/5/6 (the 3 isBlocked / auto-reset tests). Updated Tests 1/2/3/7 against the locked reference values. Added a new packets-row visibility test. Kept Test 8 (token usage) unchanged.
