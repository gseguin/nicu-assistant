## Cross-Phase Integration

- **Phase 21 → Phase 22:** SegmentedToggle now renders in both calculators; NumericInput changes don't touch toggle code paths. Clean separation.
- **Phase 22 → Phase 23:** NumericInput advisory uses `slide` transition; Phase 22 stubbed `Element.animate` + `matchMedia` in `src/test-setup.ts` which unblocked Phase 23's result-pulse tests (they rely on the same jsdom polyfills). Unintentional win.
- **Phase 23 → Phase 24:** Phase 23's `.animate-result-pulse` entrance animation is mid-flight when axe runs, but Phase 20's `no-transition` + 250ms settle pattern (still in use) freezes transitions before analysis. No contrast false positives.
- **v1.5 identity tokens → v1.6 SegmentedToggle:** Active segment picks up `var(--color-identity)` automatically via the `.identity-morphine` / `.identity-formula` route wrappers defined in v1.5. Zero prop plumbing.
- **v1.5 hero identity → v1.6 result-pulse:** Shared animation class applied on top of identity-themed hero background. Animation and color are independent concerns — clean.

## Tech Debt (Non-blocking)

1. **Magnification $effect deviation (Phase 21).** Executor fell back to MutationObserver instead of the planned top-level `$effect`. MutationObserver pattern is pre-existing v1.2 code and should still fire. Needs manual smoke test before closing as "zero risk."
2. **jsdom stubs for matchMedia + Element.animate (Phase 22).** Slightly broader than the feature work suggests but necessary for transition tests to run in jsdom. Playwright covers real browser behavior.
3. **Formula calcKey wrapped in untrack() (Phase 23).** Correct per Svelte 5 semantics; worth a comment so future refactors don't strip it.
4. **No per-phase VERIFICATION.md files.** Same as v1.5 — execute-phase orchestrator did not run the verification step. Mitigation: test suites green.
