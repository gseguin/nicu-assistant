# Phase 2 ‚Äî UI Design Contract

> Visual and interaction contract for the PERT calculator core (Oral + Tube-Feed + Safety). Generated by gsd-ui-researcher, verified by gsd-ui-checker. Consumed verbatim by gsd-planner and gsd-executor.

> **Phase 2 fills the body of the existing `<PertCalculator />` placeholder and lands a new `<PertInputs />` consumed by both the desktop sticky aside and the mobile `<InputDrawer>` already wired in `src/routes/pert/+page.svelte`. The route shell, identity hue, AboutSheet, and state singleton are Phase-1-frozen ‚Äî Phase 2 must not edit them beyond consuming.**

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom ‚Äî SvelteKit 2 + Svelte 5 + Tailwind 4 baseline, established before shadcn was an option) |
| Preset | not applicable (no `components.json`, no shadcn registry ‚Äî design tokens live in `src/app.css` and `DESIGN.json`) |
| Component library | none (in-house shared library at `src/lib/shared/components/`, plus `bits-ui` for `Slider.Root` only) |
| Icon library | `@lucide/svelte` (already pinned; no other icon packs) |
| Font | Plus Jakarta Sans (`ui-sans-serif, system-ui, sans-serif` fallbacks). Five fixed roles: Display 44 / Title 22 / Body 16 / UI 13 / Label 11. |
| Color system | OKLCH-only. `.identity-pert` purple/violet hue 285 ‚Äî Phase-1-shipped, axe-verified, not modified in Phase 2. |
| Theme | Light + dark, both designed (warm slate dark base; auto-inversion forbidden). |
| Touch floor | 48px every interactive element (WCAG + clinical). |
| Container width | `max-w-lg` mobile / `max-w-6xl` desktop, set on the route at `src/routes/pert/+page.svelte:40`. Phase 2 does not change this. |

**Project hard constraints (do NOT relax):**
- Identity-Inside Rule ‚Äî `.identity-pert` only on hero / focus rings / eyebrows / active nav. **Phase 2 must NOT extend identity-purple to chrome (segmented toggle wrapper, advisory cards, range-error text, secondary-output cards).**
- Tabular-Numbers ‚Äî every clinical number gets `class="num"`.
- Eyebrow-Above-Numeral ‚Äî hero numerals always get an UPPERCASE Label-11 eyebrow above; unit suffix sits Body/UI weight beside or below, never above.
- 11px Floor ‚Äî `text-2xs` (11px) is for labels only. Advisory copy, error messages, paragraphs step up to `text-ui` (13px) minimum.
- Red-Means-Wrong with one named carve-out per calculator. **PERT's named carve-out is the max-lipase 10,000 units/kg/day cap (PERT-SAFE-01).** No other red.
- OKLCH-only. No `#hex`, no RGB, no HSL.
- Em-dash banned in user-rendered strings, `aria-label`, screen-reader copy. Use period, colon, semicolon, parentheses. (Note: copy strings already in `pert-config.json` use a non-ASCII `‚Äî` ‚Äî see Copywriting Contract for the audit; any em-dashes that survive into rendered copy are a Phase 4 / Phase 1 latent bug, not a Phase 2 introduction.)

## Spacing Scale

The project uses Tailwind 4's default spacing rem scale (multiples of 4px). Phase 2 inherits ‚Äî no new tokens.

| Token | Tailwind class | Computed | Phase 2 usage |
|-------|----------------|----------|----------------|
| 1 | `gap-1` / `p-1` / `space-y-1` | 4px | Inline-icon nudges only |
| 2 | `gap-2` / `p-2` | 8px | Eyebrow-to-numeral gap, icon-to-text within a card row |
| 3 | `gap-3` / `py-3` / `px-3` | 12px | Advisory card padding-y, secondary-output row padding-y |
| 4 | `gap-4` / `p-4` / `px-4` | 16px | Standard card padding (matches `.card` utility), input list gap, between input groups |
| 5 | `px-5` / `py-5` | 20px | Hero card and InputDrawer body padding (matches existing `<HeroResult>` + InputsRecap analogs) |
| 6 | `gap-6` / `space-y-6` | 24px | Top-level layout: hero ‚Üî secondaries ‚Üî tertiary ‚Üî STOP-red advisory ‚Üî warning advisories ‚Äî the existing `<PertCalculator>` outer `<div class="space-y-6">` cadence is preserved |
| 8 | `gap-8` | 32px | Reserved ‚Äî not used in Phase 2 |

**Specific spacing rules for Phase 2:**

- The outer `<PertCalculator />` wrapper stays at `<div class="space-y-6">` (already present in the Phase 1 placeholder ‚Äî line 25). Phase 2 does NOT change this.
- Inside `<PertInputs />`, vertical gap between input groups = `gap-4` (16px). Inside a single labeled group (e.g. NumericInput + slider), the existing `RangedNumericInput` already uses `gap-2` internally; do not override.
- The SegmentedToggle is the FIRST child of `<PertInputs />`; gap to next input = `gap-4`.
- Secondary output rows in the secondaries card use `divide-y divide-[var(--color-border)]` + `px-5 py-4` per row, mirroring `FeedAdvanceCalculator.svelte:125-184`.
- Tertiary "Estimated daily total" line is a SINGLE `px-5 py-4` row inside the secondaries card OR a standalone `card` block; the planner picks based on copy fit (D-09 names tokens, not card count). Recommended: place tertiary in the SAME secondaries card as the last `divide-y` row to avoid a second card with one row.
- Advisory card vertical position: AFTER secondaries, BEFORE bottom of `<PertCalculator />` outer wrapper. STOP-red advisory FIRST (severity-DESC per D-10), then warnings.
- Mobile bottom-nav clearance is owned by `+page.svelte` and `NavShell`; Phase 2 inputs don't add bottom padding.

Exceptions: none.

## Typography

Five fixed roles per DESIGN.md `The Five-Roles-Only Rule`. Phase 2 does not introduce any `text-[Npx]` arbitrary value.

| Title | `text-title` | 22px | `font-bold` (weight 700) ‚Äî but the placeholder uses `font-black` for the eyebrow-as-title escape; Phase 2 keeps `font-bold` for SECONDARY OUTPUT VALUES per the feeds analog (`FeedAdvanceCalculator.svelte:135`) | 1.2 | Secondary output numerals: total lipase needed, total fat (g), capsules/month, lipase per dose, lipase per kg. **Hero typography contract (Phase 2 specific):**

- The empty-state hero (Phase 1 placeholder) already uses `text-title font-black tracking-tight text-[var(--color-identity)] uppercase` for the promoted "PERT" label and `text-2xs ... uppercase` for the mode qualifier ("Oral dose" / "Tube-feed dose"). Phase 2 PRESERVES this exact pattern for empty state.
- The valid-result hero adds `text-display font-black text-[var(--color-text-primary)]` `class="num"` for the capsule count, with a `text-ui font-medium text-[var(--color-text-secondary)]` unit suffix ("capsules" or "capsules/dose" / "capsules/day").
- Eyebrow above the numeral, identity-purple, UPPERCASE: `"PERT"` (always) + a mode-specific qualifier in `text-2xs` secondary-text weight: `"ORAL DOSE"` (oral mode) or `"TUBE-FEED DOSE"` (tube-feed mode). Both fit the existing escape-hatch shape from PertCalculator.svelte:33-46. The promoted "PERT" `text-title font-black` MUST be used to keep visual continuity with the Phase 1 empty-state ‚Äî Phase 2 just adds the numeral row below.
- Unit copy: oral hero shows "capsules/dose" (not "per dose" ‚Äî match xlsx wording). Tube-feed hero shows "capsules/day". Both `text-ui font-medium`, secondary text color.

**Secondary output typography contract:**

Each secondary output row mirrors `FeedAdvanceCalculator.svelte:127-144`:
- Eyebrow: `class="text-2xs font-semibold tracking-wide text-[var(--color-identity)] uppercase"` ‚Äî identity-purple per Identity-Inside Rule (acceptable inside a calculator-owned card).
- Numeral: `class="num text-title font-bold text-[var(--color-text-primary)]"` ‚Äî primary text, tabular.
- Unit: `class="text-ui text-[var(--color-text-secondary)]"`.
- Right-aligned ratio caption (where applicable): `class="num text-ui text-[var(--color-text-tertiary)]"`.

**Tertiary (D-09 ‚Äî Oral mode only) typography:**

Per D-09 verbatim: `text-2xs uppercase` eyebrow + `text-base` value with `class="num"`. Smaller than secondary outputs (which are `text-title`). The eyebrow color is identity-purple (matches the secondary outputs, since both live inside the calculator-owned secondaries surface). No italics.

**Advisory typography:**

- STOP-red advisory message: `text-ui font-bold text-[var(--color-error)]` (bold for clinical urgency; D-04 calls it "bold red message text"). Icon at `size={20}` is `OctagonAlert` colored `text-[var(--color-error)]`. The card background stays neutral (`bg-[var(--color-surface-card)]` ‚Äî hospital-card surface, NOT error-tinted).
- Neutral warning advisory message: `text-ui font-semibold text-[var(--color-text-primary)]` ‚Äî matches `FeedAdvanceCalculator.svelte:321` verbatim. Icon at `size={20}` is `AlertTriangle` colored `text-[var(--color-text-secondary)]`. Card background `bg-[var(--color-surface-alt)]` per the existing feeds pattern.

**Empty-state typography:**

Per D-08, the empty-state hero copy renders inside the `<HeroResult>` `children` snippet. The empty-state message is `text-ui text-[var(--color-text-secondary)]` (matches Phase 1 PertCalculator.svelte:47). Phase 2 PRESERVES this exactly when required inputs are missing.

**Banned in Phase 2:**
- `italic` (D-09 explicitly forbids on tertiary; project-wide DESIGN.md does not allow italics on clinical numbers). No `<em>` either.
- `text-xs` (12px) ‚Äî not in the five-roles set. The existing NumericInput uses `text-xs` internally for the range hint at line 188; that is INHERITED, not Phase-2-introduced. Do not add new `text-xs` in `<PertInputs />` or `<PertCalculator />`.
- `text-sm` ‚Äî not in the five-roles set (NumericInput labels at `text-ui` are correct; `text-sm` would cross-reference `text-base` minus 2px which is non-standard). Banned for Phase 2.
- Arbitrary `text-[14px]` etc. ‚Äî banned by `Five-Roles-Only Rule`.
