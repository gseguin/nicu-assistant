## Resolution architecture (the fix shape)

| Option | Description | Selected |
|--------|-------------|----------|
| Add `onValueChange` callback prop to SelectPicker (KI-1 Option 1) | SelectPicker emits explicit value-change events. PertInputs writes directly to `pertState.current.*` from the callback. No string proxy, no bidirectional race. Affects all 4 callers (feeds, gir, uac-uvc, pert). | |
| `$derived`-backed binding wrapper (KI-1 Option 2) | Wrap `bind:value` with a `$derived` getter (state → string) and a setter (string → state). Single race-free bridge. Local change to PertInputs.svelte only; SelectPicker contract untouched. | ✓ |
| Fork SelectPicker into per-calculator `<PertSelectPicker />` (KI-1 Option 3) | Internal click handler writing directly to `pertState.current.*`. Higher local complexity; isolates pert from the shared component. Last-resort. | |

**Auto-mode choice:** Option 2 — `$derived`-backed binding wrapper.
**Rationale:** Smallest blast radius, no shared-component change, eliminates the bidirectional race at the source. Explicitly endorsed by user in ROADMAP.md Phase 3.1 entry's "Resolution recommendation" line.
**Locked as:** D-01

| Option | Description | Selected |

**Locked as:** D-04
