<!-- SelectPicker dialog locator semantics (from RESEARCH §E2E Mechanics + Pitfall 6) -->

- The trigger `<button>` is INSIDE the desktop aside or mobile drawer scope. Use scope-relative: `scope.getByRole('button', { name: /^Medication/ })`. The `^` anchor prevents matching unrelated buttons (e.g., a generic 'Open menu' button that contains 'Med' substring).
- The dialog `<dialog>` is rendered at the document body root (top-level modal). Use page-level: `page.getByRole('option', { name: /^Creon$/ })`. The `^...$` anchor prevents matching `Creon (12,000 units)` if the strength label leaks into the medication picker by accident.
- For the searchable Formula picker, the search input is `<input type="text" aria-label="Filter Formula">`. Use `page.getByRole('searchbox')` OR `page.getByRole('textbox', { name: /Filter Formula/ })`. Default: `page.getByRole('searchbox')` (matches existing 8 tests' style of role-based locators).
