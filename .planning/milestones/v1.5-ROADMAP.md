## Key Accomplishments

1. **Searchable Formula picker** — case-insensitive label+manufacturer filter, ArrowDown/ArrowUp traversal between input and listbox, Enter-to-select-single-match, "No matches" state, query reset on reopen. Wired via opt-in `searchable` prop on the shared SelectPicker; Morphine pickers untouched.
2. **Per-tab visual identity** via new `--color-identity` CSS token — Morphine reuses Clinical Blue 220, Formula gets a new Teal ~195 (light + dark). Wired to exactly 4 surfaces: result hero card, focus-visible outlines, section eyebrow labels, and the active calculator tab indicator in both mobile and desktop nav.
3. **NavShell per-tab identity** via a new `identityClass` field on `CalculatorEntry` in `registry.ts`, applied directly to each tab `<a>` element — solves the "nav lives above the route" cascade trap so each tab carries its own identity without route-wrapper inheritance.
4. **Real WCAG failure caught and fixed.** Phase 19 unintentionally shipped a 3.61:1 contrast violation: the new Morphine schedule eyebrow text on the new identity-hero card (which reused `--color-accent-light` as background). Phase 20's axe sweep caught it and tuned `.identity-morphine` light hero to literal `oklch(95% 0.04 220)`, clearing 4.5:1. `--color-accent*` and shell chrome stayed untouched.
5. **Playwright a11y suite extended** — focus-ring rendered + dark-mode results-visible variants now covered for both calculators. 8/8 axe sweeps green. No `disableRules` escape hatches.
6. **App version** finally bumped from `1.2.0` (stuck since v1.2) to `1.5.0` so the about dialog reflects the shipped milestone.

## Phase Details

### Phase 18: Searchable SelectPicker
**Goal:** Clinicians using the Formula picker can filter 30+ formulas by typing and reach the right option in under a second via keyboard or touch, while Morphine pickers remain unchanged.
**Requirements:** SEARCH-01..06, A11Y-03

Plans:
- [x] 18-01-PLAN.md — SelectPicker `searchable` prop, filter, keyboard handlers, "No matches"
- [x] 18-02-PLAN.md — Wire Formula picker + T-12..T-18 component tests

Key choices:
- Conditional `role="combobox"` ARIA so existing tests T-01..T-11 stay green
- `showGrouped = hasGroups && !isSearching` bypass for the two-level grouped rendering when actively searching
- `searchQuery` reset in BOTH `openPicker` and the dialog `close` handler
- `$effect` clamps `focusedIndex` when `filteredOptions.length` shrinks

### Phase 19: Tab Identity Token
**Goal:** Each calculator tab carries a distinct but restrained visual identity via a single `--color-identity` token wired to exactly four surfaces, so clinicians instantly recognize which calculator they are in without the shell or body chrome changing color.
**Requirements:** IDENT-01..07

Plans:
- [x] 19-01-PLAN.md — Identity tokens in `app.css`, registry `identityClass`, NavShell per-tab wiring, route wrappers
- [x] 19-02-PLAN.md — Wire identity to body surfaces (heroes, eyebrows, focus-visible)

Key choices:
- `app.css` changes were purely additive (zero deletions, no v1.4 a11y regression risk)
- Wrapper class `.identity-morphine` / `.identity-formula` on the route page outer `<div>` — rejected data-attributes and per-route `+layout.svelte` (don't currently exist)
- Morphine identity reuses `var(--color-accent)`; Formula identity = new Teal 195
- BMF Amber tokens left untouched (forward-looking, currently unused in markup) — IDENT-06 satisfied automatically
- Shell chrome (AboutSheet, DisclaimerModal, title bar, theme toggle) untouched — IDENT-07 enforced via grep guards

### Phase 20: Identity A11y Verification
**Goal:** Every surface using `--color-identity` meets WCAG 2.1 AA contrast in both themes for both tabs, verified by the existing axe-core sweep with color-contrast enabled.
**Requirements:** A11Y-01, A11Y-02

Plans:
- [x] 20-01-PLAN.md — Extend axe sweeps (focus ring + dark visible) and conditionally tune identity OKLCH

Key findings:
- Existing v1.4 sweeps already cover light+dark for both tabs via `withTags(['wcag2a','wcag2aa'])` (color-contrast enabled transitively)
- Two real gaps closed: (a) light-mode "results visible" tests didn't render the focus ring; (b) dark-mode "results visible" variants never analyzed the new dark hero OKLCH values
- Caught and fixed Phase 19's hidden 3.61:1 contrast failure by tuning Morphine light hero to `oklch(95% 0.04 220)`
- Final result: 8/8 Playwright axe sweeps green, no rule suppression
