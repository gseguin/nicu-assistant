# Milestone v1.4 — UI Polish Requirements

**Goal:** Broad polish pass across both calculators and the shell — visual refinement, mobile UX, animation/feedback, accessibility deep-dive — guided by `.impeccable.md` as the design contract.

## Validated

### Picker (PICK) — shared SelectPicker rewrite

- [x] **PICK-01**: User sees a custom-styled `<dialog>`-based modal picker (replacing bits-ui Select) wherever a SelectPicker is rendered
- [x] **PICK-02**: User can open the picker, navigate options with keyboard (Arrow/Home/End/Enter/Esc), and the trigger refocuses on close
- [x] **PICK-03**: User sees grouped options (e.g. formulas grouped by manufacturer) rendered with the new picker style
- [x] **PICK-04**: User sees focus-visible outlines and selected-state styling that match `.impeccable.md` (warm clinical, OKLCH tokens, no hardcoded colors)
- [x] **PICK-05**: User on mobile sees the picker fill the available width with comfortable touch targets (≥48 px) and safe-area-aware bottom padding
- [x] **PICK-06**: No `bits-ui` `Select` imports remain anywhere in `src/`. Full `bits-ui` dependency removal deferred — `AboutSheet.svelte` and `DisclaimerModal.svelte` still consume `bits-ui` `Dialog` and will be ported in a later milestone phase.

### Fortification calculator (FORT)

- [x] **FORT-01**: User on mobile sees "Target Calorie" and "Unit" pickers laid out on the same row
- [x] **FORT-02**: User sees the "Amount to Add" hero card restyled to match morphine wean's result theming (typographic hierarchy, surface treatment, brand accent)
- [x] **FORT-03**: User sees consistent spacing, typography, and empty-state styling across the Fortification calculator after the polish pass

### Morphine wean calculator (MORPH)

- [x] **MORPH-01**: User sees the morphine wean calculator pass the same visual refinement sweep (spacing, typography, focus states, empty/loading states)
- [x] **MORPH-02**: User sees the morphine wean calculator pick up the new shared SelectPicker without regressions

### Shell & navigation (SHELL)

- [x] **SHELL-01**: User sees a polished title bar (info + theme buttons) consistent with `.impeccable.md`
- [x] **SHELL-02**: User sees a polished bottom tab bar / top nav with refined active-state, hover, and focus styling
- [x] **SHELL-03**: User sees a polished theme toggle and About sheet with consistent typography and motion

### Animation & feedback (ANIM)

- [x] **ANIM-01**: User sees micro-animations (focus transitions, recalc affordances, hero updates) tuned for calm, not flashy
- [x] **ANIM-02**: User with `prefers-reduced-motion: reduce` sees all non-essential motion disabled or reduced to instant transitions

### Accessibility deep-dive (A11Y)

- [x] **A11Y-01**: User sees all interactive surfaces meet WCAG 2.1 AA contrast in both light and dark themes after a high-contrast review pass
- [x] **A11Y-02**: Screen reader user can navigate both calculators and the shell with correct labels, live regions, and reading order
- [x] **A11Y-03**: Keyboard-only user can reach every interactive surface, with visible focus and predictable tab order
- [x] **A11Y-04**: axe-core e2e suite passes for both calculators in light + dark after the polish pass
