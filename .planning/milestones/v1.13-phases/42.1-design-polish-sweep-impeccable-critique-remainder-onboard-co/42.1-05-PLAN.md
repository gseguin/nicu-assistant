# Plan 42.1-05: Hero-Fills-Viewport + Sticky Inputs Drawer (Structural Redesign, QA-Flag Gated)

## Objective

Implement D-08's structural redesign: on every calculator route, the hero card fills the top 60% of the viewport on first paint with pre-computed default values, and the inputs collapse to a sticky drawer above the bottom nav that the user expands by tap. Ships behind a localStorage feature flag (`nicu_assistant_drawer_ux=on`) per RESEARCH risk #1 — gives 2 weeks of internal QA before flipping the default. The flag-OFF path renders the existing inputs-above-hero shell unchanged. Ships in Wave 5 because it depends on `<HeroResult>` (Plan 2); flipping the default to ON is a separate future PR (out of scope for this plan).

`autonomous: false` — this is the most invasive UX change in the phase. Recommend a user checkpoint after the drawer component lands but before the 5 routes are migrated, so the user can see the pattern in one route and provide feedback before propagation.

## Threat Model

| Threat | Severity | Mitigation | Verified by |
|--------|----------|------------|-------------|
| QA flag bypass surface (`nicu_assistant_drawer_ux=on`) — a user could discover and enable the unfinished UX accidentally | low | The flag is informational and OFF-by-default in localStorage. Users discovering it would see a different layout, not a broken or insecure one. The flag-ON path is fully tested (all axe + e2e specs gated by addInitScript) | InputDrawer.test.ts + hero-drawer.spec.ts both cover the flag-ON path; flag-OFF path uses the existing tests |
| Drawer state persists across navigation in a way that conflicts with sessionStorage form state | low | Drawer state (`expanded`) is component-local; calculator state (weight, dose) is unaffected and persists per existing per-calculator sessionStorage | E2E test 4: navigate from /morphine-wean to /uac-uvc, drawer state may reset to collapsed (acceptable); calculator state persists |
| Hero pre-computed defaults expose stale values on slow devices | low | Defaults are baked into the calculator state at module load (existing pattern in `state.svelte.ts` files); HeroResult renders synchronously | Component tests for the hero already assert default-state output; Plan 5 doesn't change calculator state |
| InputDrawer's bits-ui Dialog vs native `<dialog>` choice — bits-ui is heavier but better tested for focus trap | low | Use bits-ui Dialog for consistency with AboutSheet (after Plan 3 hoist). bits-ui Dialog is already in the bundle | Bundle size check — no new deps added |
| Plan 5 lands before Plan 6's residue cleanup, breaks SegmentedToggle move | low | Plan 5 wraps inputs (including SegmentedToggle) in InputDrawer's children Snippet — toggle position inside inputs is preserved either way | Plan 6 T1 (SegmentedToggle move) operates on the inputs content tree, not the wrapper — runs cleanly after Plan 5 |

    Create `src/lib/shared/components/InputDrawer.svelte` per UI-SPEC + PATTERNS §"D-08" guidance:

    Recommended approach: native `<dialog>` (matches HamburgerMenu's pattern — built-in focus trap + Esc + scrim). Collapsed state shows the 1-line summary above the bottom nav with a chevron-up handle; expanded state opens the dialog over the hero.

    ```svelte
    <script lang="ts">
      import type { Snippet } from 'svelte';
      import { ChevronUp, ChevronDown } from '@lucide/svelte';

      let {
        summary,
        expanded = $bindable(false),
        children
      }: {
        summary: string;
        expanded?: boolean;
        children: Snippet;
      } = $props();

      let dialog = $state<HTMLDialogElement | null>(null);

      $effect(() => {
        if (!dialog) return;
        if (expanded && !dialog.open) dialog.showModal();
        if (!expanded && dialog.open) dialog.close();
      });

      function handleClose() {
        expanded = false;
      }
    </script>

    <!-- Collapsed handle: pinned above bottom nav -->
    <button
      type="button"
      class="fixed right-0 left-0 z-20 flex min-h-14 items-center justify-between gap-3
             border-t border-[var(--color-border)] bg-[var(--color-surface-card)]/95 backdrop-blur
             px-4 py-2 text-ui font-medium text-[var(--color-text-primary)]
             focus-visible:outline-2 focus-visible:outline-offset-[-2px] focus-visible:outline-[var(--color-accent)]
             md:hidden"
      style="bottom: calc(env(safe-area-inset-bottom, 0px) + 4rem);"
      aria-label="Open inputs drawer"
      aria-expanded={expanded}
      onclick={() => (expanded = true)}
    >
      <span>{summary}</span>
      <ChevronUp size={20} aria-hidden="true" />
    </button>

    <dialog
      bind:this={dialog}
      class="input-drawer-dialog bg-[var(--color-surface-card)] text-[var(--color-text-primary)] shadow-2xl"
      onclose={handleClose}
    >
      {#if expanded}
        <div class="flex h-full flex-col" style="padding-bottom: env(safe-area-inset-bottom, 0px)">
          <header class="flex items-center justify-between border-b border-[var(--color-border)] px-5 py-4">
            <h2 class="text-base font-semibold">Inputs</h2>
            <button
              type="button"
              class="icon-btn min-h-[48px] min-w-[48px]"
              aria-label="Close inputs drawer"
              onclick={() => (expanded = false)}
            >
              <ChevronDown size={20} aria-hidden="true" />
            </button>
          </header>
          <div class="flex-1 overflow-y-auto p-4">
            {@render children()}
          </div>
        </div>
      {/if}
    </dialog>

    <style>
      .input-drawer-dialog {
        margin: 0;
        padding: 0;
        border: 0;
        position: fixed;
        bottom: 0;
        left: 0;
        right: 0;
        top: auto;
        width: 100%;
        max-width: 100%;
        height: 80vh;
        height: 80dvh;
        max-height: 80vh;
        max-height: 80dvh;
        border-top-left-radius: 1rem;
        border-top-right-radius: 1rem;
      }
      .input-drawer-dialog::backdrop { background: var(--color-scrim); }
      @media (prefers-reduced-motion: no-preference) {
        .input-drawer-dialog[open] { animation: slide-up 200ms cubic-bezier(0.22, 1, 0.36, 1); }
        .input-drawer-dialog[open]::backdrop { animation: fade-in 200ms ease; }
      }
      @keyframes slide-up { from { transform: translateY(100%); } to { transform: translateY(0); } }

    </style>
    ```
