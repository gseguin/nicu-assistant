# Plan 42.1-04 Summary

## What changed

**D-16 — Dock magnification removed from both calculators.** The macOS-Dock-style scroll-driven `transform: scale(1+t)` + `box-shadow` boost on mobile cards was deleted from `MorphineWeanCalculator.svelte` (lines 11-96 of the prior file) AND from `GlucoseTitrationGrid.svelte` (lines 25-110). Both included a MutationObserver, requestAnimationFrame scroll handler, MAX_SCALE / MAX_SHADOW_BOOST constants, and bind:this refs.

The morphine schedule's active-step affordance is now driven by tap: each row carries `role="button"`, `tabindex="0"`, an Enter/Space keyboard handler, and toggles `aria-current="step"` + a `ring-1 ring-inset ring-[var(--color-identity)]` class on the active row. The GIR titration grid needed no replacement: its existing roving-tabindex `selectedBucketId` already drives the active-row UX via `aria-checked` + `bg-[var(--color-identity-hero)]` + inset ring.

**D-17 — CSS theme transition narrowed.** The global `html:not(.no-transition) *` selector (which paints every element on theme switch — expensive on slow devices) was replaced with an explicit 8-selector list: `body, .card, button, [role="tab"], [role="dialog"], input, select, [data-theme-aware]`. The transition declaration was kept verbatim (`background-color 200ms ease-out, color 200ms ease-out, border-color 200ms ease-out`).

**D-18 — Verified (no code change).** `animate-result-pulse` (app.css:166-184) and the HamburgerMenu drawer slide (HamburgerMenu.svelte:174-181) both already gate on `prefers-reduced-motion`. Verified by Read; no edit needed.

**D-19 — Static root redirect with pathname gate.** The JS `goto('/morphine-wean', { replaceState: true })` in `+page.svelte` was replaced with a static-shell-level redirect: `src/app.html` now carries an inline `<script>` that does `if (window.location.pathname === '/') { window.location.replace('/morphine-wean'); }` BEFORE any other HTML parses. This runs before SvelteKit hydration, so bots, slow connections, and no-JS clients with JS see the redirect immediately. `+page.svelte` keeps a `<meta http-equiv="refresh">` + `<link rel="canonical">` + the same pathname gate + `<noscript>` for the documentation contract.

**Why app.html instead of `<svelte:head>`:** with `ssr = false` in `src/routes/+layout.ts`, SvelteKit cannot inject `<svelte:head>` markup into the prerendered HTML before hydration. Adapter-static also overrides `build/index.html` with its SPA fallback shell. The PLAN's failure-handling section explicitly authorizes the app.html approach.

**Pathname gate strategy:** Initially the gate tried to remove the meta-refresh tag from the DOM when `pathname !== '/'`. This raced with Chromium's meta-refresh queueing and yielded an infinite redirect loop in e2e. The fix: gate the `location.replace` call directly on `pathname === '/'`. No meta to remove → no race.
