# Plan 42.1-02: HeroResult Shared Component + 5-Calculator Refactor + Decimal Precision Summary

Morphine display gains 3-decimal/2-decimal precision via formatMg/formatPercent (calculations.ts byte-identical — D-21 honored). UAC/UVC regains the directional icons (ArrowDownToLine / ArrowUpFromLine) lost in the v1.13 cleanup pass.**

## Deviations from Plan

### Deviation: morphine schedule region keeps its own aria-live

- **Plan T4 acceptance:** `grep -F 'aria-live="polite"' src/lib/morphine/MorphineWeanCalculator.svelte` returns 0 matches.
- **Actual:** the schedule `<section aria-label="Weaning schedule" aria-live="polite" aria-atomic="true">` (the per-step list, not the summary chip) was preserved with its own aria-live. The summary chip's wrapping aria-live was removed (HeroResult owns it now).

Removing it would silently regress the screen-reader experience for the per-step output. The plan acceptance was stricter than necessary — the spec intent is "no DUPLICATE live regions in the same subtree" not "no aria-live anywhere in the file."
- **Verified:** the e2e spec `'schedule has aria-live region for screen readers'` (`morphine-wean.spec.ts:70-73`) still passes — assertion intact.
- **Impact:** the file has 1 `aria-live="polite"` reference (line ~207). New unit test `'hero is now a HeroResult consumer (single .animate-result-pulse <section>)'` asserts there is exactly ONE `.animate-result-pulse <section>` in the rendered output (HeroResult's).
