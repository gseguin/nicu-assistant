Land the `formatMg`/`formatPercent` 3-decimal/2-decimal display utilities and apply them to morphine. Restore the D-05 third cue (`ArrowDownToLine`/`ArrowUpFromLine`) on UAC/UVC. ## Threat Model

| Threat | Severity | Mitigation | Verified by |
|--------|----------|------------|-------------|
| HeroResult `value` prop is a string and could permit XSS via `{@html}` | low | The component renders `value` via plain `{value}` interpolation (text node, auto-escaped by Svelte). Confirmed by reading PATTERNS §1 reference excerpt and DESIGN.json snippet — neither uses `{@html}` | HeroResult.test.ts scenario asserts `.textContent` equality on a value string containing `<script>` tag literal renders as text |
| `formatMg(NaN)` or `formatMg(Infinity)` returning `'NaN'` or `'Infinity'` to a clinical hero | medium | Defensive `if (!Number.isFinite(value)) return '0.000'` at top of formatMg + formatPercent | format.test.ts covers `NaN`, `Infinity`, `-Infinity` returning `'0.000'` / `'0.00%'` |
| Refactor accidentally drops aria-live from a calculator's hero | medium | HeroResult itself owns `aria-live="polite" aria-atomic="true"` on the outer `<section>` — calculators that previously had aria-live on their wrapping `<div>` MUST remove it (PATTERNS §"HeroResult adoption pattern" step 4 — duplicate live regions degrade SR announcements) | Component tests for each refactored calculator assert ONE `[aria-live="polite"]` element in the hero subtree, not zero, not two |
| Parity tests break because formatMg changes the underlying calculation precision | low | D-21 forbids touching calculations.ts; formatMg is a render-boundary string formatter — calculation Number values stay full-precision | `git diff src/lib/morphine/calculations.ts` → empty; existing parity tests untouched |
| New eyebrow string contains em-dash by mistake | low | UI-SPEC explicitly mandates middle-dot U+00B7 for UAC/UVC eyebrows; `commit 917ecf2` already purged em-dash systemwide | grep `'—'` and `'--'` across modified .svelte files returns no matches in user-rendered strings |

    ```ts
    /**
     * Format a milligram value as a 3-decimal string for display.
     * Display formatting only — never used inside calculation code (D-21).
     */
    export function formatMg(value: number): string {
      if (!Number.isFinite(value)) return '0.000';
      return value.toFixed(3);
    }

    /**
     * Format a percentage value as a 2-decimal string with % suffix.
     */
    export function formatPercent(value: number): string {
      if (!Number.isFinite(value)) return '0.00%';
      return value.toFixed(2) + '%';
    }
    ```
