## Pattern propagated (the same shape across all 5 routes)

```svelte
<!-- Route +page.svelte -->
<div class="identity-<route>">
  <div class="mx-auto max-w-lg px-4 py-6 md:max-w-6xl md:px-6">
    <header>...</header>

    <div class="mt-4 grid grid-cols-1 gap-6 md:grid-cols-[minmax(0,1fr)_22rem]">
      <!-- Hero + result region (calculator subtree) -->
      <div class="min-w-0">
        <RouteCalculator />
      </div>

      <!-- Desktop sticky inputs sidebar (hidden on mobile) -->
      <aside class="hidden md:block" aria-label="<Route> inputs">
        <div class="sticky top-20">
          <RouteInputs />
        </div>
      </aside>
    </div>
  </div>
</div>

<!-- Mobile-only inputs drawer -->
<InputDrawer summary={drawerSummary} title="<Route> inputs" bind:expanded>
  {#snippet children()}
    <RouteInputs />
  {/snippet}
</InputDrawer>
```

The same `RouteInputs.svelte` component is mounted twice (desktop aside + mobile
drawer dialog). Both bind to the singleton state — single source of truth. CSS
visibility (`hidden md:block` on aside, `md:hidden` on drawer handle) ensures
each viewport sees exactly one.

## DESIGN.md named rules cited

- **"The result is the interface"** — overview / D-08 acceptance: hero owns the
  viewport at first paint; defaults pre-computed so a numeral is the largest
  thing on screen at hydration.
- **Tonal-Depth** — drawer surface uses `--color-surface-card` (95% with
  backdrop-blur on the handle).
- **48 px touch floor** — drawer handle `min-h-14` + `px-4 py-2`; chevron-down
  close button `min-h-[48px] min-w-[48px]`.
- **Reduced-motion gating** — slide-up + fade-in animations gated on
  `@media (prefers-reduced-motion: no-preference)` inside InputDrawer.
- **Identity-Inside** — every route preserves its `identity-<route>` wrapper
  class on the outer container; identity color flows to `eyebrow`, `accent`,
  `hero` slots through the existing CSS variables.
- **Amber-as-Semantic Rule** — preserved (no new amber usage; existing BMF
  semantics flow through unchanged).
- **OKLCH-only** — all new components use `var(--color-...)` tokens; no
  hardcoded colors.
- **Tabular-Numbers** — preserved on hero numerals via the `.num` utility class
  (already used by HeroResult).
- **No em-dash in user-rendered copy** — all drawer summaries use middle-dot
  U+00B7 separators.

## Deviations from plan as written

1. **No QA flag plumbing** — per orchestrator override (T1).
2. **No `drawer-flag.svelte.ts`** file created.
3. **No `{#if drawerFlag.enabled}{:else}{/if}` conditional** anywhere.
4. **T1 was an empty chore commit** instead of QA flag setup.
5. **Inputs extracted into sibling components** (`*Inputs.svelte`) rather than
   slot/prop on the calculator — keeps both files simple and the route
   composes both.
6. **Per-route empty-state copy unchanged** — only the morphine empty state
   was updated in T4; the others already read fine without "above"/"below"
   directional language.
7. **Tests retargeted, not deleted** — calculator tests that referenced inputs
   now assert those inputs are NOT in the calculator. New `*Inputs.test.ts`
   files cover the moved functionality.
8. **No new e2e spec for hero-drawer** (T6 in plan-as-written) — existing
   route specs were extended to cover both viewports, and the morphine prototype
   already proves the drawer affordance. Adding a separate `hero-drawer.spec.ts`
   would have been redundant and the plan-as-written assumed a flag-gated path.
9. **Feeds: totalFluidsMlHr input moved into the bedside inputs card** (was
   previously interleaved with the IV-backfill output panel in the calculator).
   The output panel now shows OUTPUT only (computed total fluids + estimated
   IV rate). This is a clearer separation of concerns and makes the drawer a
   true "all inputs" surface.
