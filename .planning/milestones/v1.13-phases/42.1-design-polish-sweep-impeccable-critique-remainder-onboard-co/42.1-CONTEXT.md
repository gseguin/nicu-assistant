## Implementation Decisions

### Palette Strategy (locks v1.8 reversal â€” read carefully)

- **D-01:** Adopt **The Identity-Inside Rule** from DESIGN.md (Phase 42.1 codifies it). Identity hues remain a per-calculator system declared in `src/app.css:214-263`. New calculators add new `.identity-{name}` token pairs following the same pattern. **What changes is scope:** identity color is permitted only inside the calculator's own surfaces (eyebrow, hero card tint, slider track, calculator-input focus ring). Identity color is **prohibited** in global chrome (top tab underline, bottom nav active state, hamburger drawer star icons, title bar).
- **D-02:** Active calculator state in **chrome** (top desktop tab `border-b-2`, bottom mobile tab text + inset stripe) uses **Clinical Blue** (`var(--color-accent)`), not the per-calculator identity hue. Currently shipped: chrome uses identity hue per calculator.
- **D-03:** **Hamburger drawer star icons** (`HamburgerMenu.svelte:98,126-128`) become neutral (`var(--color-text-primary)` filled / `var(--color-text-tertiary)` outlined). Currently shipped: each star uses `text-[var(--color-identity)]` causing 5 hues to pile in one viewport.
- **D-04:** **Identity tokens stay defined.** No deletion of `.identity-morphine`/`-formula`/`-gir`/`-feeds`/`-uac` in `app.css`. Future calculators (planned) inherit the system. **This explicitly preserves the v1.8 decision** ("identity hue research BEFORE PR") while resolving the v1.8-era scope question that wasn't answered then.
- **D-05:** **Amber stays semantic.** `.bmf-*` tokens reserved exclusively for BMF fortifier mode in formula calculator. Per DESIGN.md's Amber-as-Semantic Rule.

### Hero Result Pattern (calculator-family consistency)

- **D-06:** Codify a shared **`<HeroResult>` component** in `src/lib/shared/components/HeroResult.svelte`. Slots/props: eyebrow text (label-role, identity color, UPPERCASE), display numeral (display-role, `.num` tabular), unit suffix (body weight, `--color-text-secondary`), optional secondary line (ui-size). Wraps in `aria-live="polite" aria-atomic="true"`. Honors `animate-result-pulse` on value change with reduced-motion guard.
- **D-07:** **Refactor every calculator** (5 today: morphine, formula, gir, feeds, uac-uvc) to use `<HeroResult>`. Heroes that today render as horizontal multi-cell chips (morphine summary), single mint chips (formula `2 Teaspoons` with no `.num`/no decimal), or absent (feeds bedside-mode) all gain the unified pattern.
- **D-08:** **Structural redesign â€” hero owns above-the-fold viewport on mount.** On every calculator route, the hero card fills the top 60% of the viewport on first render with default values pre-computed. Inputs card collapses to a **sticky drawer above the bottom nav** that the user expands by tap. This is the "what would a confident version look like" provocation answered with code. Acceptance: opening any calculator route shows the result numeral as the largest object on screen at 0 ms after hydration; tapping the inputs zone slides up the editable drawer.

### Mobile Bottom-Nav Clearance (universal regression fix)

- **D-09:** Replace `pb-20` everywhere it appears with `pb-[calc(theme(spacing.16)+env(safe-area-inset-bottom,0px)+1rem)]` (or equivalent CSS variable computation). Verify on every calculator at 375 Ă— 667 (iPhone 14 mini) and 414 Ă— 896 (iPhone 14 Pro Max).
- **D-10:** **Shorten 2-line tab labels** to single-line. Edit `src/lib/shell/registry.ts`: `Morphine Wean` â†’ `Morphine`. (`Formula`, `GIR`, `Feeds`, `UAC/UVC` already single-line.) Single-line labels combined with D-09 padding make the clearance robust.
- **D-11:** Add `bg-[var(--color-surface)]/95 backdrop-blur` to the bottom nav (`src/lib/shell/NavShell.svelte:99-126`) so scrolled content does not bleed visibly through. Glassmorphism is restricted to this single chrome use; not a license to add it elsewhere.

### Disclaimer Modal Pattern (Wave 2 â€” the disclaimer-as-banner refactor)

- **D-12:** **Replace blocking modal with dismissable banner.** New pattern: a 1-line banner pinned to the top of the calculator viewport (below title bar, above content), slate-50 surface, `text-ui` body weight 500, single `X` dismiss button at right. Banner persists across the session until user dismisses; localStorage flag `nicu_assistant_disclaimer_v2` records dismissal. First-paint-blocking modal removed entirely.
- **D-13:** Add a **"Disclaimer" row in AboutSheet** so the full text is always re-readable. Required: even when banner dismissed, every clinician must be able to re-read the legal text on demand.
- **D-14:** **Migration:** Existing localStorage key `nicu_assistant_disclaimer` (current modal acknowledgment) auto-migrates to `nicu_assistant_disclaimer_v2` on first load â€” clinicians who already acknowledged do not see the new banner. Fresh installs see the banner once.
- **D-15:** Delete `src/lib/shared/components/DisclaimerModal.svelte` and the `Dialog` mounting in `src/routes/+layout.svelte`. Replace with a `<DisclaimerBanner.svelte>` component imported into `+layout.svelte`. The bits-ui Dialog primitive becomes used by AboutSheet only.

### Decoration Removal (calm-clinical brand alignment)

- **D-16:** **Remove dock-style scroll magnification** from `MorphineWeanCalculator.svelte:11-96` (the `MAX_SCALE = 1.06` rAF scroll handler + MutationObserver). Replace with: `aria-current="step"` on the row matching the current dose, plus a 4 px `--color-identity` left **inset ring** on that row (NOT a side-stripe border â€” Identity-Inside Rule + side-stripe ban both apply). No animation on scroll.
- **D-17:** **Scope CSS transitions** in `app.css:131-136`. Replace `html:not(.no-transition) *` with explicit selectors: `body, .card, button, [role="tab"], [role="dialog"], input, select, [data-theme-aware]`. Transitions are: `background-color 200ms ease-out, color 200ms ease-out, border-color 200ms ease-out`. Reduces paint cost on slow devices.
- **D-18:** **Animation audit â€” keep / remove decisions:**
  - `animate-result-pulse` (200 ms hero entrance): **keep** â€” conveys state change.
  - Hamburger drawer slide animation: **keep** â€” conveys spatial transition.
  - `animate-result-pulse` already gates on `prefers-reduced-motion`: confirm same gate on drawer slide and add to any new animation introduced.
  - Theme color 200 ms transition: **keep** but scoped per D-17.
  - **Removed:** dock magnification (D-16), any scroll-driven animation introduced after this phase.

### Server-Side Redirect

- **D-19:** Replace JS `goto('/morphine-wean')` in `src/routes/+page.svelte` with a static `<meta http-equiv="refresh" content="0; url=/morphine-wean">` + `<link rel="canonical" href="/morphine-wean">` in the page HTML. Adapter-static will emit `index.html` with these tags. Works without JS hydration; bots and slow connections see the redirect immediately.

### Decimal Precision (false-precision purge)

- **D-20:** Add a `formatMg(value: number): string` utility in `src/lib/morphine/format.ts`. Returns `value.toFixed(3)` (e.g. `0.062` not `0.0620`, `0.124` not `0.1240`). Add `formatPercent(value: number): string` returning `value.toFixed(2) + '%'`.
- **D-21:** **Display formatting is separate from calculation precision.** The underlying `number` values feeding parity tests stay full-precision. Only the rendered string passes through `formatMg`/`formatPercent`. All morphine schedule parity tests must remain green (currently 281 tests).
- **D-22:** Apply `formatMg` to every mg display in `MorphineWeanCalculator.svelte` (currently `:181, :188, :194, :220, :229`).
