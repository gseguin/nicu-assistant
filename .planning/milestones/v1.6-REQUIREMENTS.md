**Milestone goal:** Resolve the "two patterns for one decision" inconsistency between calculators, and harden the input layer so out-of-range values can't slip through silently.

## v1.6 Requirements

### Shared SegmentedToggle

- [x] **TOG-01**: Shared `SegmentedToggle` component in `src/lib/shared/components/` with API consistent with `SelectPicker`.
- [x] **TOG-02**: Active segment uses `var(--color-identity)` — picks up per-tab identity automatically.
- [x] **TOG-03**: Keyboard nav ←/→/Home/End with `role="tablist"` + `role="tab"` ARIA matching Morphine's prior pattern.
- [x] **TOG-04**: Morphine consumes the shared component for Linear/Compounding, inline tablist code removed.
- [x] **TOG-05**: Formula Base uses `SegmentedToggle` instead of `SelectPicker`.
- [x] **TOG-06**: Existing morphine + fortification tests still pass with no behavior change.

### NumericInput Hardening

- [x] **HARD-01**: `NumericInput` accepts optional `min` and `max` props.
- [x] **HARD-02**: Visible range hint (`0.1–200 kg` en-dash format) rendered below the field when `min`/`max` supplied. Optional `showRangeHint` prop (default `true`) allows per-field opt-out.
- [x] **HARD-03**: Blur-gated "Outside expected range — verify" advisory (em dash U+2014) in `var(--color-error)`. No auto-clamp.
- [x] **HARD-04**: Advisory clears immediately when value returns to range.
- [x] **HARD-05**: All Morphine + Formula NumericInput ranges sourced from `morphine-config.json` / `fortification-config.json` `inputs` block via typed TS wrappers.
- [x] **HARD-06**: Component tests cover hint rendering, blur-outside message, blur-back-inside clear, and no-clamp guarantee (13 tests T-01..T-13).

### Result Feedback

- [x] **FEED-01**: Both result heroes carry `aria-live="polite"` + `aria-atomic="true"`.
- [x] **FEED-02**: Shared `.animate-result-pulse` class in `src/app.css` — 200ms scale-from-95% entrance, gated by `@media (prefers-reduced-motion: reduce)`.
- [x] **FEED-03**: No auto-scroll, no focus theft — verified by spy assertions in both test files.

### Accessibility

- [x] **A11Y-01**: Every v1.6 surface passes WCAG 2.1 AA in both themes via axe-core sweep. 12/12 Playwright tests green, zero OKLCH tuning required.
- [x] **A11Y-02**: New v1.6 surfaces (toggle, range hint, advisory) meet 4.5:1 text / 3:1 non-text contrast in both themes.
- [x] **A11Y-03**: SegmentedToggle keyboard interactions covered by component tests (T-03..T-05).

## Outcomes

All 18 requirements shipped as defined. One post-ship adjustment landed in the same milestone:

- **HARD-02 extension (84ceae9):** Added an optional `showRangeHint` prop (default `true`) after the user noted the `1–1000 mL` hint on the Formula Starting Volume field felt redundant. The clinical range is still enforced (advisory on blur still fires) but the visible hint is suppressed on that specific field. Zero impact on other call sites.

## Out of Scope (v1.6)

- Replacing `SelectPicker` itself — it stays for the Formula picker (30+ options) and other N-of-many choices.
- Auto-clamping out-of-range values — advisory message only, never overwrite user input.
- New calculators.
- Server-side validation or remote range catalog.
- Reset / undo affordances.
