## Formula Reference (from research + requirements)

Let `grams = gramsToAdd` (the raw Excel output in grams before unit conversion).

**General formula (CALC-02):**
```
grams = (volume Ã— (targetKcal âˆ’ baseKcal)) / (29.57 Ã— calConc âˆ’ dispFactor Ã— targetKcal)
```
where `baseKcal = 20` if `base === 'breast-milk'`, else `0`.

**Unit conversion (CALC-03):** `amountToAdd` is the grams figure transformed per unit:
- `grams` â†’ `grams`
- `scoops` â†’ `grams / grams_per_scoop`
- `teaspoons` â†’ `grams / 2.5`
- `tablespoons` â†’ `grams / 7.5`
- `packets` â†’ special case (CALC-04)

**Packets special case (CALC-04):**
- If formula is NOT Similac Human Milk Fortifier â†’ `amountToAdd = 0`
- If HMF and `targetKcalOz === 24` â†’ `amountToAdd = volume / 25`
- If HMF and `targetKcalOz === 22` â†’ `amountToAdd = volume / 50`
- Else (HMF with other target) â†’ `amountToAdd = 0`

**BM + Teaspoons shortcut (CALC-05):**
- If `base === 'breast-milk'` AND `unit === 'teaspoons'`:
  - targetKcalOz 22 â†’ `amountToAdd = (volume / 90) Ã— 0.5`
  - targetKcalOz 24 â†’ `amountToAdd = (volume / 90) Ã— 1`
  - other targetKcalOz â†’ fall through to general formula

Order of precedence: Packets special case > BM+Tsp shortcut > general formula with unit conversion.

**Yield + grams-added helper (CALC-06 â€” B-4 resolution):**

`gramsAdded` (the gram mass of what was actually added, used by both yield and exact kcal/oz) is computed by a **single per-unit helper** â€” NOT by inverse-solving the general formula, and NOT from a hard-coded HMF packet mass constant:

```typescript
// Pseudocode â€” executor writes this as a module-private helper.
function gramsAdded(amountToAdd: number, unit: UnitType, formula: FortificationFormula): number {
  switch (unit) {
    case 'grams':       return amountToAdd;
    case 'scoops':      return amountToAdd * formula.grams_per_scoop;
    case 'packets':     return amountToAdd * formula.grams_per_scoop;  // same as scoops â€” see note below
    case 'teaspoons':   return amountToAdd * 2.5;
    case 'tablespoons': return amountToAdd * 7.5;
  }
}
```

**Why packets == scoops for `gramsAdded`:** The xlsx Calculator tab does not store an HMF "packet mass" constant. The yield formula (xlsx G10) for the Packets branch computes `gramsAdded = packets Ã— VLOOKUP(formula, A:D, 4) = packets Ã— grams_per_scoop`. For Similac HMF, `grams_per_scoop = 5` (xlsx row B29:D29), so one "packet" is treated as one scoop = 5 g. This is the ground truth â€” no new constant, no new JSON field.

Then:
```
yieldMl = volumeMl + gramsAdded Ã— displacement_factor
```

**Exact kcal/oz (CALC-07):**
```
exactKcalPerOz = (baseKcal Ã— volume / 29.57 + gramsAdded Ã— calConc) / (yieldMl / 29.57)
```

**Suggested Starting Volume (CALC-08):**
1. Round `amountToAdd` to the nearest whole unit (e.g. 2 tsp).
2. Back-calculate the volume that would produce exactly that whole-unit amount using the inverse of whichever formula path was taken (general, BM+Tsp shortcut, or HMF Packets).
3. **Rounding lock (per M-3):** `mL = Math.round(suggestedV)` (integer mL). `oz = (Math.round((suggestedV / 29.57) * 10) / 10).toFixed(1)` (one decimal).
4. **Format lock (per M-4):** `"<mL> (<oz> oz)"`. Example: `"180 (6.1 oz)"`.
5. **Invalid / zero lock (per M-4):** If `amountToAdd` is 0, `volumeMl <= 0`, or any computed field is non-finite, return `{ amountToAdd: 0, yieldMl: 0, exactKcalPerOz: 0, suggestedStartingVolumeMl: '0 (0 oz)' }` â€” matching the spreadsheet's `IF(G9=0, "0 (0 oz)", ...)` behavior.

(Researcher manually verified the documented Neocate case to full precision â€” use `toBeCloseTo(23.5101662125341, 4)` for the exact kcal/oz assertion.)
