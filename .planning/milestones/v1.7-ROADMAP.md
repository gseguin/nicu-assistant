## Key Accomplishments

1. **Label cleanup** — Starting Volume and Target Calorie lose their unit parentheticals. The suffix already communicates the unit; labels now read `"Starting Volume"` and `"Target Calorie"`.
2. **Single-row layout** — Formula picker (wider) and Starting Volume (narrower) share one row via `flex gap-3 items-end`, `flex-1 min-w-0` on Formula, `basis-28 shrink-0` on Volume. Target Calorie + Unit moved to their own `grid-cols-2` row below. Works at all screen sizes.
3. **Auto-select packets** — Picking a packets-capable formula now auto-switches the Unit to `packets`. Extended the existing v1.3 `$effect` with a second branch so both directions (auto-reset off packets when leaving a packets-capable formula, auto-select when entering one) live in one place. Data-driven via a new `packetsSupported?: boolean` field on `FortificationFormula` + a `formulaSupportsPackets(id)` helper — no hardcoded formula IDs.
4. **`showRangeError` prop** on `NumericInput` — symmetric complement to v1.6's `showRangeHint` opt-out. Formula Starting Volume now passes both opt-outs, so the surface is clean (no hint, no advisory) while the clinical range is still enforced for the scroll-wheel clamp.
5. **Version bump to 1.7.0**.

## Phase Details

### Phase 25: Formula Micro-Polish
**Goal:** Three small rough edges in the Formula calculator fixed — labels stop repeating units, Formula + Starting Volume share a row, and packets auto-select when the formula supports them.
**Requirements:** LBL-01..03, ROW-01..03, AUTO-01..04, REG-01..03

Executed as a quick task — no formal PLAN.md file. Executor was briefed inline from REQUIREMENTS.md and completed all three edits in one pass (plus the `showRangeError` follow-up when the user requested hiding the Formula advisory).

Key choices:
- Packets support moved into config as `packetsSupported?: boolean` rather than deriving from a fictional `supportedUnits` array (no such field existed)
- Auto-packets effect uses the same `prevFormulaId` guard as the v1.3 auto-reset, so both branches share a single `prevFormulaId !== currFormulaId` gate — avoids double effects
- Target Calorie + Unit moved into their own `grid-cols-2` row to balance the layout after Formula + Volume took the top row
- `showRangeError` added as a non-breaking opt-out (default `true`)

Deviation: two Formula advisory a11y tests (added in v1.5 Phase 24) had to be removed because the only Formula NumericInput (Starting Volume) now opts out of the advisory. Coverage preserved by Morphine's equivalent tests (weight/dose/percent still render the advisory in both themes).
