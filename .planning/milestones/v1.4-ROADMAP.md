Drop-in compatible — `FortificationCalculator.svelte` and `package.json` untouched. New `--color-scrim` OKLCH token for `<dialog>::backdrop`. jsdom `HTMLDialogElement` polyfill with setup-time self-test and BLOCKER fix for dialog `aria-labelledby` collision with the trigger. 11 new colocated unit tests including a regression guard (T-11).
2. **Fortification visual polish** — `grid grid-cols-2` pairs Target Calorie + Unit on mobile; "Amount to Add" hero restyled with `bg-[var(--color-accent-light)]`, `text-5xl` tabular bold numeric, separate unit label span — adopting morphine wean result theming. `text-2xs` eyebrow unification across hero and Verification cards; calmer `gap-3` rhythm. Zero logic/state/config edits.
3. **Morphine Wean polish** — step card eyebrows and meta typography bumped to `text-2xs font-semibold uppercase tracking-wide`, matching Phase 13 language. Dock-style scroll magnification untouched. MORPH-02 vacuously satisfied (morphine calculator has no SelectPicker consumers).
4. **Shell & navigation polish** — title bar gains `min-h-14` (56 px bedside thumb room) and `tracking-tight` app name; desktop calculator tab anchors get `focus-visible:outline-[var(--color-accent)] rounded-t-lg`; mobile bottom tab bar gets `min-h-14`, `transition-colors`, visible accent focus outline, active state weight bump (`font-semibold`), and hover affordance on inactive items.
5. **Animation & reduced-motion audit** — every motion surface accounted for. Closed the one remaining gap (`NumericInput` `transition:slide` on the error message) by reading `matchMedia('(prefers-reduced-motion: reduce)')` at module load and passing `duration: 0` when reduced. Full audit table in `14-01-SUMMARY.md` (sic — Phase 16 SUMMARY). No layout-shift animations anywhere in the app.
6. **Accessibility deep-dive** — probed dark-mode axe-core color-contrast sweep and identified 5 violations traced to two under-bright tokens. Bumped `--color-text-secondary: oklch(70% → 80% 0.010 228)` and `--color-accent: oklch(72% → 82%, chroma 0.14 → 0.12, 220)` inside the Clinical Blue family. Removed the `disableRules(['color-contrast'])` escape hatch from both a11y specs and added a `no-transition` class + 250 ms settle before axe analyze so the probe reads final computed colors, not 200 ms theme-transition interpolations. All 6 Playwright a11y tests now green with color-contrast enabled in both themes.

## Phase Details

### Phase 12: Shared SelectPicker Rewrite
**Goal**: Both calculators consume a custom-styled `<dialog>`-based modal picker that replaces the bits-ui Select, matches `.impeccable.md`, and drops bits-ui Select imports (full bits-ui removal deferred).
**Requirements**: PICK-01, PICK-02, PICK-03, PICK-04, PICK-05, PICK-06 (PICK-06 rescoped mid-phase to "no bits-ui Select imports remain in src/"; full bits-ui removal deferred because `AboutSheet` and `DisclaimerModal` still consume bits-ui `Dialog`)
**Success Criteria**:
1. Every SelectPicker instance in the app opens a custom `<dialog>`-based modal picker — no bits-ui Select UI is rendered anywhere
2. A keyboard user can open the picker, move through options with Arrow/Home/End, confirm with Enter, dismiss with Esc, and the trigger button receives focus again on close
3. Grouped options (e.g. formulas grouped by manufacturer) render correctly with the new picker style, and focus-visible/selected states use `.impeccable.md` OKLCH tokens with no hardcoded colors
4. On a mobile viewport the picker fills the available width with ≥48 px touch targets and honors `env(safe-area-inset-bottom)`
5. No `bits-ui` `Select` imports remain anywhere in `src/`; the app still builds and passes tests
**Plans**: 1 (12-01) — Atomic `<dialog>`-based rewrite + scrim token + jsdom polyfill + unit tests

**Requirements**: ANIM-01, ANIM-02
**Success Criteria**:

2. With `prefers-reduced-motion: reduce` set, all non-essential motion is disabled or reduced to instant transitions
3. Dock-style scroll magnification on the morphine wean schedule either disables or degrades gracefully under reduced motion
4. No animation introduces a layout shift after first paint
**Plans**: 1 (16-01) — NumericInput slide reduced-motion guard + full audit

### Phase 17: Accessibility Deep-Dive
**Goal**: Both calculators and the shell pass a full WCAG 2.1 AA deep-dive covering contrast, screen readers, keyboard, and an automated axe-core sweep in light and dark
**Requirements**: A11Y-01, A11Y-02, A11Y-03, A11Y-04
**Success Criteria**:
1. A high-contrast review pass documents that every interactive surface meets WCAG 2.1 AA contrast in both light and dark themes
2. A screen reader user (VoiceOver or NVDA) can navigate both calculators and the shell with correct labels, live regions, and a predictable reading order
3. A keyboard-only user can reach every interactive surface with visible focus and predictable tab order — no focus traps, no hidden targets
4. The Playwright axe-core e2e suite passes for both calculators in light and dark modes with zero violations
**Plans**: 1 (17-01) — Dark-mode contrast token bumps + a11y spec cleanup + no-transition settle
