# Plan 24-01 — A11y Verification for v1.6 surfaces

## Goal

Verify every v1.6 surface meets WCAG 2.1 AA contrast in both themes for both tabs, via the existing Playwright + axe-core sweep. Tune OKLCH only if something fails. No rule suppression.

## Surfaces introduced by v1.6 that need verification

1. **SegmentedToggle active segment** (Morphine Clinical Blue + Formula Teal, light + dark) — `text-[var(--color-identity)]` on `bg-[var(--color-surface-card)]` (from verbatim lift of Morphine pattern).
2. **SegmentedToggle inactive segment** — `text-[var(--color-text-secondary)]` on `bg-[var(--color-surface-alt)]` container.
3. **NumericInput range hint** — `text-[var(--color-text-tertiary)]` on `bg-[var(--color-surface-card)]`.
4. **NumericInput advisory message** — `text-[var(--color-error)]` on `bg-[var(--color-surface-card)]`.
5. **Result hero animation** — no new colors, but the axe sweep must still pass while the pulse is mid-flight (Phase 20 pattern uses `no-transition` + 250ms settle — verify it still suffices).

## Task 1 — Extend axe sweeps to render the new v1.6 states

Both `e2e/morphine-wean-a11y.spec.ts` and `e2e/fortification-a11y.spec.ts` already run axe in light + dark + results-visible + focus-rendered variants. Ensure the new states are rendered during at least one variant per theme:

- The **SegmentedToggle** is rendered in both calculators as soon as the page loads — already in every sweep automatically.
- The **range hint** is rendered as soon as any NumericInput with `min`/`max` appears — already in every sweep.
- The **advisory message** requires triggering blur with an out-of-range value. Add a new test variant per spec file: fill an input with a clearly out-of-range value (e.g. weight = 0 in Morphine, volume = 9999 in Formula), blur, apply the existing `no-transition` + 250ms settle pattern, run axe.

### Action

For each of `e2e/morphine-wean-a11y.spec.ts` and `e2e/fortification-a11y.spec.ts`, add a new test:

```ts
test('{calculator} advisory message has no axe violations in light mode', async ({ page }) => {
  await page.goto('/{route}');
  await page.getByRole('button', { name: /got it/i }).click().catch(() => {});

  // Fill an input with an out-of-range value and blur
  const input = page.getByRole('spinbutton').first();
  await input.fill('99999');
  await input.blur();

  // Wait for advisory to render
  await expect(page.getByText(/outside expected range/i)).toBeVisible();

  // Existing no-transition + settle pattern
  await page.addStyleTag({ content: '*, *::before, *::after { transition: none !important; animation: none !important; }' });
  await page.waitForTimeout(250);

  const results = await new AxeBuilder({ page }).withTags(['wcag2a', 'wcag2aa']).analyze();
  expect(results.violations).toEqual([]);
});
```

And a matching dark-mode variant (toggle theme first).
