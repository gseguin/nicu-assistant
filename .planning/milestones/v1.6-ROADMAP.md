## Key Accomplishments

1. **Shared SegmentedToggle component** extracted from Morphine's Linear/Compounding tablist into `src/lib/shared/components/`. Identity-aware (active segment uses `var(--color-identity)`, auto-picking up Clinical Blue in Morphine / Teal in Formula via v1.5 route wrappers). Keyboard nav (←/→/Home/End), `role="tablist"` ARIA. Morphine refactored to consume it (zero behavior change); Formula `Base` SelectPicker replaced.
2. **NumericInput hardened** with a visible range hint (`0.1–200 kg` en-dash format), blur-gated "Outside expected range — verify" advisory (em dash), and a `showRangeHint` opt-out prop for cases where the hint is redundant. Critically: no auto-clamp — the user's typed value is preserved even when out of range.
3. **Clinical input ranges moved to JSON config** via a new `inputs` top-level key in `morphine-config.json` and `fortification-config.json`, with typed TS wrappers. Removes magic numbers from component code.
4. **Result feedback parity** — both calculator heroes now carry `aria-live="polite"` + `aria-atomic="true"` for screen reader announcement, plus a shared `.animate-result-pulse` class (`src/app.css`) providing a 200ms scale-from-95% entrance animation gated by `prefers-reduced-motion`. No auto-scroll, no focus theft.
5. **Zero WCAG regressions** — every v1.6 surface (toggle active/inactive in both identities, range hint, advisory) passed AA on first axe sweep in both themes. 12/12 Playwright a11y tests green, no OKLCH tuning required, no rule suppression.
6. **Bonus infra fix** (Phase 22): jsdom stubs for `matchMedia` + `Element.animate` in `src/test-setup.ts` — unblocked blur-triggered transition tests and carried forward to Phase 23's result-pulse tests.

## Phase Details

### Phase 21: Shared SegmentedToggle (build + wire)
**Goal:** Both calculators express 2-option choices through one shared, identity-aware, keyboard-accessible toggle.
**Requirements:** TOG-01..06, A11Y-03

Plans:
- [x] 21-01-PLAN.md — Build `SegmentedToggle.svelte` + 7 tests
- [x] 21-02-PLAN.md — Wire Morphine + Formula consumers

Key choices:
- `role="tablist"` + `role="tab"` ARIA (1:1 with Morphine's prior pattern — zero behavior risk)
- `aria-controls` dropped (Morphine's version pointed at panel IDs that didn't exist — latent bug fixed by extraction)
- Active segment uses `var(--color-identity)` via route-wrapper cascade — no prop plumbing
- `crypto.randomUUID()` for per-instance tab IDs (matches SelectPicker)

Deviation: Phase 21-02 fell back to a pre-existing `MutationObserver` for dock-magnification re-trigger on mode change, rather than the planned top-level `$effect`. Reason: `triggerMagnification` is scoped inside `onMount` as a closure-local `let`. Documented in tech debt; manual smoke test recommended before future refactors.

### Phase 22: NumericInput Hardening
**Goal:** Out-of-range clinical values cannot slip through silently, but the user keeps full typing freedom.
**Requirements:** HARD-01..06

Plans:
- [x] 22-01-PLAN.md — `hasBlurred` gate, range hint markup, config-sourced ranges, 13 tests

Key findings:
- `NumericInput` already had `min`/`max` props and `rangeError` derivation — real work was surfacing the hint, gating the error on blur, rewriting the message literal, and migrating inline ranges to JSON.
- Two existing test assertions on `"Maximum is 100"` / `"Minimum is 0"` literals were updated to the new copy.
- `Number.isFinite` guard added to the scroll-wheel increment handler since min/max became effectively optional.
- jsdom stubs for `matchMedia` + `Element.animate` added to `src/test-setup.ts` to unblock blur-transition tests.

### Phase 23: Result Feedback (aria-live + entrance)
**Goal:** When a result appears or updates, screen reader users hear it and sighted users see a calm motion cue — without losing input flow.
**Requirements:** FEED-01, FEED-02, FEED-03

Plans:
- [x] 23-01-PLAN.md — Shared `.animate-result-pulse`, Morphine aria-live, Formula `{#key calcKey}` entrance, no-scroll guard tests

Key findings:
- Each calculator had partial coverage: Morphine had the pulse animation without `aria-live`, Formula had `aria-live` without the animation. Phase 23 was a parity pass.
- Animation extracted to `src/app.css` under `@layer components` as `.animate-result-pulse` (200ms `scale(0.95)` → `scale(1)`, reduced-motion gated).
- Old `.animate-summary-pulse` class fully retired.
- Formula's `calcKey += 1` wrapped in `untrack()` to prevent Svelte `effect_update_depth_exceeded`.

### Phase 24: A11y Verification
**Goal:** Every v1.6 surface passes WCAG 2.1 AA in both themes for both tabs via axe-core.
**Requirements:** A11Y-01, A11Y-02

Plans:
- [x] 24-01-PLAN.md — Extend axe sweeps with 4 new advisory-message variants (light + dark, both calculators)

Key findings:
- Zero OKLCH tuning required — every surface passed on first sweep. Unlike v1.5 Phase 20, which caught a real 3.61:1 regression.
- Advisory message triggered by filling a NumericInput with an out-of-range value (`99999`) and blurring, then applying the v1.4 `no-transition` + 250ms settle pattern before analysis.
- 12/12 Playwright a11y tests green, no `disableRules` escape hatches.
